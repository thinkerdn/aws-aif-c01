1️⃣ なぜデータ準備がAI/MLの核心か

⏱️ 学習時間: 約15分

AI/MLモデルの性能は、アルゴリズムだけでなく「データの質」で決まります。データ準備と前処理は、信頼できるモデルを構築するための土台です。

データ準備の目的

  • データの精度と一貫性を向上させる
  • ノイズや欠損に強い入力データをつくる
  • 特徴量を適切に表現し、学習を安定化させる
  • 再現性のある処理手順を確立する

💡 Key Concept

AI/MLにおける「80/20の法則」: モデル性能の80%はデータ準備と前処理で決まり、20%がアルゴリズムの選択です。

前処理が失敗すると起きる問題

  • 欠損値が学習結果を歪める
  • 異常値にモデルが引きずられる
  • カテゴリ変数の扱いが誤ると意味を失う
  • データ漏洩が発生し、評価結果が過剰になる

2️⃣ データ収集と品質評価

⏱️ 学習時間: 約25分

データソースの種類

  1. 構造化データ: CSV、データベース、ログデータ
  2. 半構造化データ: JSON、XML、APIレスポンス
  3. 非構造化データ: テキスト、画像、音声

データ品質の主要指標

指標 説明
完全性 欠損値が少ないこと。重要な列やレコードが抜けていない状態。
正確性 実際の値と一致しているか。測定値や入力値の誤りを検出する。
一貫性 同じ意味のデータが同じ形式で揃っているか。
重複性 重複レコードや重複イベントが含まれていないか。
有効性 値の範囲や形式が定義に合っているか。

品質評価のための簡単なチェックリスト

  • 列数・行数が期待通りか
  • 欠損値の割合を確認（NULL、空文字、特殊値）
  • データ型が一貫しているか
  • 外れ値や異常値の存在を可視化
  • ラベルが正しく分布しているか（分類問題の場合）
例: 販売データの「売上金額」が負の値を含む場合、それは測定ミスやデータ入力エラーの可能性があります。

3️⃣ 欠損値・外れ値の処理

⏱️ 学習時間: 約35分

欠損値のパターン

  • Missing Completely at Random (MCAR): ランダムに欠損
  • Missing at Random (MAR): 他の変数に依存して欠損
  • Missing Not at Random (MNAR): 欠損自体に意味がある

💡 重要

欠損値処理は一律ルールではなく、ドメイン知識と欠損の原因に応じて戦略を選ぶ必要があります。

欠損値への対処法

  • 削除: 欠損行や欠損列を削除する
  • 補完: 平均・中央値・最頻値・前後値などで埋める
  • 推定: モデルや統計的手法を使う
  • フラグ付け: 欠損を示す特徴量を追加する

外れ値の検出と処理

  1. 箱ひげ図や標準偏差で検出する
  2. 分布から外れた値を確認する
  3. ビジネスルールに照らして妥当性を判断する
  4. 修正、削除、または隔離して扱う
# pandasによる欠損値確認例
import pandas as pd

df = pd.read_csv('sales_data.csv')
print(df.isnull().sum())

# 平均値補完
for col in ['price', 'quantity']:
    df[col].fillna(df[col].median(), inplace=True)

# 外れ値の簡易検出
q1 = df['price'].quantile(0.25)
q3 = df['price'].quantile(0.75)
IQR = q3 - q1
outliers = df[(df['price'] < q1 - 1.5 * IQR) | (df['price'] > q3 + 1.5 * IQR)]
print(outliers)

4️⃣ 特徴量エンジニアリングの基礎

⏱️ 学習時間: 約30分

特徴量とは

特徴量はモデルが学習する入力データの要素です。元データから価値ある情報を取り出し、モデルがより良く学習できるように整形します。

代表的な特徴量エンジニアリング

  • 数値特徴量の合成: 比率、差分、平均
  • カテゴリ特徴量の分割: 層別化、バケット化
  • 日時特徴量の展開: 年、月、曜日、時間帯
  • ドメイン固有の変換: 例) 顧客LTV、距離、頻度

カテゴリ変数の扱い

  • One-Hot Encoding: 各カテゴリを二値特徴に変換
  • ラベルエンコーディング: カテゴリを整数に変換
  • ターゲットエンコーディング: 目的変数の平均を利用
  • 頻度エンコーディング: 登場頻度を特徴量化
手法 長所 短所 One-Hot Encoding 直感的で多くのモデルで扱いやすい 高次元化しやすい ラベルエンコーディング 次元数が増えない 順序がない場合に誤解を招く ターゲットエンコーディング 高次元カテゴリを効果的に扱える データ漏洩のリスクがある
💡 ポイント: 特徴量設計では「意味のある変換」を優先し、モデルに与える値がビジネスの文脈に即しているか確認します。

5️⃣ データ変換と正規化

⏱️ 学習時間: 約25分

なぜ変換が必要か

異なるスケールの変数が混在していると、距離ベースのモデルや勾配降下法で学習が不安定になります。適切なスケーリングや変換で学習の収束を改善します。

代表的な変換手法

  1. 標準化 (Standardization): 平均0、分散1 になるよう変換
  2. 正規化 (Normalization): 0〜1 に収める
  3. 対数変換: 分布の歪みを改善する
  4. Box-Cox / Yeo-Johnson: 連続値を正規分布に近づける
from sklearn.preprocessing import StandardScaler, MinMaxScaler

scaler = StandardScaler()
df[['age', 'income']] = scaler.fit_transform(df[['age', 'income']])

minmax = MinMaxScaler()
df[['score']] = minmax.fit_transform(df[['score']])

# 対数変換
import numpy as np
df['sales_log'] = np.log1p(df['sales'])

スケーリングの選択基準

  • 線形回帰やSVM、ニューラルネットでは標準化が有効
  • ツリーベースのモデルでは必須ではない
  • 0〜1に揃える場合はMin-Max正規化を使用
  • 分布が歪んでいる場合は対数変換やBox-Coxを検討

6️⃣ テキスト・画像データの前処理

⏱️ 学習時間: 約20分

テキストデータの前処理

  • 小文字化
  • 不要な記号・HTMLタグの除去
  • トークン化
  • ストップワードの除去
  • ステミング / 形態素解析
  • Bag-of-Words / TF-IDF / 埋め込みベクトル化

前処理例

import re
from sklearn.feature_extraction.text import TfidfVectorizer

text = "今日は、AIの学習を始めます。"
text = text.lower()
text = re.sub(r'[^\w\s]', '', text)

vectorizer = TfidfVectorizer(token_pattern=r'(?u)\b\w+\b')
X = vectorizer.fit_transform([text])
print(vectorizer.get_feature_names_out())

画像データの前処理

  • リサイズ
  • 正規化
  • データ拡張 (回転、反転、切り抜き)
  • 色空間変換
  • バッチ化
注意: 画像の前処理では、訓練データのみデータ拡張を行い、検証/テストデータは補正のみとします。これにより、評価が過学習の影響を受けにくくなります。

7️⃣ データパイプラインと再現性

⏱️ 学習時間: 約20分

前処理の再現性を保つ

同じデータに同じ処理を繰り返せることは、AI/MLワークフローの基本です。前処理をコードやパイプラインにまとめ、バージョン管理します。

パイプラインの構成要素

  • データ読み込み
  • データ検査・クレンジング
  • 欠損値・外れ値の処理
  • 特徴量変換
  • 訓練用・評価用データの分割

AWSで使える管理手法

  • AWS Glue: ETLパイプラインの自動化
  • Amazon SageMaker Data Wrangler: 前処理の可視化とエクスポート
  • Amazon S3: データの永続化とバージョン管理
  • AWS Step Functions: ワークフロー連携
💡 ベストプラクティス: データ前処理は「手動処理」ではなく、「自動化されたパイプライン」に落とし込むべきです。再現性と監査性を高めます。

8️⃣ 演習: 具体的な前処理フロー

⏱️ 学習時間: 約20分

ケース: 顧客離反予測

顧客データには、数値、カテゴリ、テキスト、日付が混在しています。次の手順で前処理を設計します。

ステップ1: データ品質の確認

  1. 欠損値の割合を確認
  2. カテゴリ列のユニーク数を確認
  3. 数値列の分布と外れ値を可視化

ステップ2: 欠損値の扱い

  • 年齢: 中央値で補完
  • 職業: 'Unknown' などのカテゴリにまとめる
  • 最終購入日: 日付差から新しい特徴を作成

ステップ3: 特徴量変換

  • 購買回数 / 継続期間 = 平均購入頻度
  • ログ変換で購入額の裾を改善
  • 曜日や月を特徴量化

ステップ4: モデル入力への変換

  • カテゴリ変数をOne-Hot化
  • 数値特徴量を標準化
  • テキストコメントをTF-IDF変換

9️⃣ ベストプラクティスとAWS活用例

⏱️ 学習時間: 約10分

前処理のベストプラクティス

  • データのサンプルを可視化し、異常を早期に発見する
  • 前処理を細かいステップに分けて検証する
  • トレーニングセットとテストセットの処理は同一の手順にする
  • 特徴量を説明できるようにコメントやドキュメントを残す

AWSでの具体例

  • Amazon SageMaker Data Wrangler でデータ可視化と加工を行う
  • AWS Glue でデータカタログを作成し、ETLを定期実行する
  • S3に前処理済みデータを保存し、SageMakerで再利用する
  • Step FunctionsでGlue、SageMaker、Lambdaを連携し、運用パイプラインを構築する
