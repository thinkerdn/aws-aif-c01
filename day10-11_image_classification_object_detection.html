1️⃣ 画像分類と物体検出の概要

⏱️ 学習時間: 約20分

画像分類は画像全体をカテゴリに分類するタスク、物体検出は画像内の複数オブジェクトの位置と種類を検出するタスクです。両者は画像分析の基本であり、多くの実務で利用されます。

画像分類 vs 物体検出

特徴 画像分類 物体検出
目的 画像全体を1つのラベルに分類 画像内の各オブジェクトを検出・識別
出力 カテゴリラベル ラベル + バウンディングボックス
応用例 顔認識、医療画像診断 防犯カメラ、物流のバーコード検出
💡 ポイント: 画像分類は「何が写っているか」、物体検出は「どこに何があるか」を答えるタスクです。

2️⃣ 画像分類の基本と代表的手法

⏱️ 学習時間: 約30分

代表的な手法

  • 畳み込みニューラルネットワーク(CNN)
  • 転移学習(ResNet、EfficientNet、MobileNet)
  • Fine-tuningによるモデルの適応

転移学習のメリット

  • 小規模データセットでも高精度を出しやすい
  • 学習時間を大幅に短縮できる
  • 事前学習モデルの特徴抽出能力を利用できる
例: ImageNetで事前学習したResNetを使い、自社製品の画像分類モデルを数時間で構築することが可能です。

3️⃣ 物体検出の基本と代表的手法

⏱️ 学習時間: 約30分

代表的な物体検出モデル

  • R-CNN / Fast R-CNN / Faster R-CNN
  • YOLO (You Only Look Once)
  • SSD (Single Shot MultiBox Detector)
  • EfficientDet

検出アルゴリズムの違い

  • R-CNN系: 高精度だが遅い
  • YOLO系: 低レイテンシだが小物体検出が難しい場合がある
  • SSD系: バランスの良い速度と精度

💡 重要

物体検出では、候補領域の生成とクラス判定を同時に行う手法がリアルタイム推論に向きます。

4️⃣ SageMakerでの画像モデル構築

⏱️ 学習時間: 約30分

SageMakerでの画像モデル構築手順

  1. 画像データをS3に保存し、トレーニング/検証/テストに分割
  2. 前処理やオーグメンテーションを定義
  3. Estimatorで学習ジョブを作成
  4. 学習済みモデルを保存し、エンドポイントをデプロイ

転移学習の実装例

from sagemaker.pytorch import PyTorch

estimator = PyTorch(
    entry_point='train.py',
    role=role,
    framework_version='1.13',
    py_version='py39',
    instance_type='ml.p3.2xlarge',
    instance_count=1,
    hyperparameters={
        'epochs': 10,
        'batch-size': 32,
        'learning-rate': 0.0001
    }
)
estimator.fit({'train': train_input, 'validation': val_input})

学習スクリプトの構成要素

  • データセットのロードと前処理
  • モデルの定義と転移学習
  • 損失関数と最適化手法の設定
  • 評価とモデル保存

5️⃣ Amazon RekognitionおよびVision AIの利用

⏱️ 学習時間: 約20分

Amazon Rekognitionの機能

  • 画像分類: ラベル検出、顔認識、テキスト検出
  • 物体検出: 物体・シーンの検出
  • カスタムラベル: 自社ドメインの物体検出モデル

Rekognition Custom Labelsの利点

  • トレーニングデータをアップロードして独自ラベルを学習
  • ラベル付け済みデータを使った自動学習
  • モデルデプロイがAWSにより管理される
💡 注意: Rekognitionは低コストで高速に導入できる一方、複雑なモデル調整が必要なケースではSageMakerを併用するのが有効です。

6️⃣ データ拡張と前処理のポイント

⏱️ 学習時間: 約20分

画像前処理の一般手順

  • リサイズ・クロップ
  • ピクセル値の正規化
  • データ形式の統一 (JPEG/PNG → テンソル)

データ拡張の代表例

  • 回転、平行移動、拡大縮小
  • 左右反転・色変換
  • ランダムクロップやノイズ付加
例: 物体検出ではバウンディングボックスの座標もデータ拡張に応じて変換する必要があります。

7️⃣ 評価指標とモデル改善の戦略

⏱️ 学習時間: 約25分

画像分類の評価指標

  • Accuracy
  • Precision / Recall / F1スコア
  • 混同行列
  • ROC AUC

物体検出の評価指標

  • mAP (mean Average Precision)
  • IoU (Intersection over Union)
  • Precision-Recall Curve

💡 重要

物体検出では、IoUの閾値を調整することで検出の厳しさを制御できます。用途に応じてしきい値を最適化しましょう。

モデル改善の戦略

  • データ拡張で汎化性能を高める
  • 転移学習で事前学習モデルを活用する
  • クラス不均衡を補正する
  • 学習率やバッチサイズを調整する

8️⃣ 実践ワークフロー: 画像分類/物体検出の構築

⏱️ 学習時間: 約20分

実践手順

  1. データ収集・ラベル付け
  2. データ形式をS3/COCO/Pascal VOCに変換
  3. 前処理・オーグメンテーションを定義
  4. 学習ジョブを実行し、ネットワークを訓練
  5. 検証データで性能を評価・改善
  6. モデルをデプロイし、推論を確認

注意すべきポイント

  • データの多様性が不足すると過学習しやすい
  • 小さな物体は検出精度が低くなりがち
  • 画像サイズとバッチサイズのバランスを調整する

9️⃣ 運用とデプロイの注意点

⏱️ 学習時間: 約15分

デプロイの考慮点

  • リアルタイム推論かバッチ推論かを用途で選ぶ
  • 推論レイテンシとコストのトレードオフを評価する
  • モデルのバージョン管理とロールバックを準備する

運用監視

  • 予測精度の劣化を定期的にチェック
  • 推論データのドリフトを検出する
  • エラーや異常な入力のログを取得する
💡 運用のポイント: 画像モデルは入力画像の条件に敏感です。照明・解像度・撮影角度の違いを運用時に考慮しましょう。
