2-1 AIの基本的な概念と用語

AIの基本用語

AWS AIF-C01 では、AI・ML分野の基本用語を正確に理解していることが問われます。 以下の用語と定義を確実に押さえましょう。

用語 定義・説明 試験ポイント
人工知能 (AI) 人間の知能をコンピューターで模倣する技術の総称。推論・学習・問題解決・認識などを含む。 MLやDLはAIの一部（サブセット）
機械学習 (ML) 明示的にプログラムせずに、データからパターンを学習するAIの手法。 DLはMLのサブセット
ディープラーニング (DL) 多層（深い）ニューラルネットワークを使った機械学習。画像・音声・自然言語処理に強い。 大量データとGPUが必要
ニューラルネットワーク 脳の神経回路を模倣した計算モデル。入力層・隠れ層・出力層で構成される。 層を深くしたのがDL
基盤モデル (Foundation Model) 大量の汎用データで事前学習した大規模モデル。様々なタスクにファインチューニングして使用可能。 Amazon Bedrockで利用
大規模言語モデル (LLM) テキスト生成・理解に特化した大型基盤モデル。GPT-4、Claude、Titanなどが該当。 LLMは基盤モデルのサブセット
パラメータ モデルの学習によって調整される重み（数値）。GPT-4は推定1.8兆パラメータ。 パラメータ数 ∝ モデル性能・コスト
トークン LLMがテキストを分割する最小単位。英語では約0.75単語/トークン、日本語は1〜2文字/トークン程度。 入出力コストの計算単位

推論の種類

AIシステムが結論を導き出す推論の形式を理解しておきましょう。

推論の種類 説明 AIでの応用例
演繹的推論
Deductive		 一般原則・ルールから具体的な結論を導く。「すべてのAはBである。XはAである。∴XはBである。」 ルールベースAI、エキスパートシステム
帰納的推論
Inductive		 多数の具体例から一般的な法則・パターンを導く。機械学習の基礎となる考え方。 機械学習全般（訓練データからパターン学習）
統計的推論
Statistical		 確率・統計に基づいて不確実な状況下で判断を下す。 ベイズ分類器、確率的モデル
連想推論
Associative		 関連するパターン・類似性に基づいて推測する。「Aが起きた状況でBも起きやすい」。 推薦システム、協調フィルタリング

機械学習に使うデータの種類とデータ分割

データの種類

データ種類 説明・形式 対応するAWS/アルゴリズム
構造化データ テーブル形式で整理されたデータ（CSV、RDB など）。行と列で管理可能。 XGBoost、Linear Learner、Amazon Redshift
非構造化データ テキスト、画像、音声、動画など定型フォーマットを持たないデータ。全データの約80%。 Deep Learning、Rekognition、Comprehend、Transcribe
半構造化データ JSON、XML、ログファイルなど、スキーマはあるが完全なテーブル形式ではないデータ。 AWS Glue、Amazon Athena
時系列データ タイムスタンプ付きのシーケンシャルデータ（センサー値、株価、売上推移 など）。 Amazon Forecast、DeepAR（SageMaker組み込み）

ラベルデータ vs 非ラベルデータ

  • ラベルデータ（アノテーション済みデータ）: 各データに正解ラベル（目的変数）が付与されているデータ。例: メールに「スパム/非スパム」のラベルが付いている。教師あり学習に必要。
  • 非ラベルデータ: 正解ラベルがないデータ。教師なし学習や自己教師あり学習で活用。大量に存在するため、LLMの事前学習はこのデータを利用。
試験ポイント: ラベリングは高コスト。少量のラベルデータ + 大量の非ラベルデータを活用するのが「半教師あり学習」。

学習の種類

教師あり学習 (Supervised Learning)

  • 定義: ラベル付きデータ（入力と正解のペア）でモデルを訓練する
  • タスク:
    • 分類（Classification）: カテゴリを予測。例: スパム判定、画像分類
    • 回帰（Regression）: 連続値を予測。例: 価格予測、需要予測
  • 代表例: スパムメール検出、医療診断、株価予測、顔認識
  • AWSアルゴリズム: XGBoost、Linear Learner、Factorization Machines、Image Classification

教師なし学習 (Unsupervised Learning)

  • 定義: ラベルなしデータから自律的にパターンや構造を発見する
  • タスク:
    • クラスタリング（Clustering）: 類似データのグループ分け
    • 次元削減（Dimensionality Reduction）: 特徴量の圧縮・可視化
    • 異常検知（Anomaly Detection）: 通常パターンから外れたデータの検出
  • 代表例: 顧客セグメンテーション、不正取引検知、レコメンデーション
  • AWSアルゴリズム: K-Means（クラスタリング）、PCA（次元削減）、Random Cut Forest（異常検知）

強化学習 (Reinforcement Learning)

  • 定義: エージェントが環境と相互作用し、試行錯誤を通じて累積報酬を最大化する学習
  • 構成要素:
    • エージェント（Agent）: 意思決定を行う主体
    • 環境（Environment）: エージェントが相互作用する外部システム
    • 状態（State）: 現在の環境の状況
    • 行動（Action）: エージェントが選択できる操作
    • 報酬（Reward）: 行動の結果として得るフィードバック
  • 代表例: ゲームAI（AlphaGo）、ロボット制御、広告入札最適化、推薦システム
  • AWS: Amazon SageMaker RL（Ray RLlib統合）

半教師あり学習 (Semi-Supervised Learning)

  • 定義: 少量のラベルデータと大量の非ラベルデータを組み合わせて学習する
  • メリット: ラベリングのコストを大幅に削減しつつ、高い性能を維持できる
  • 活用例: 医療画像診断（ラベリングに専門家が必要）、Webコンテンツ分類

自己教師あり学習 (Self-Supervised Learning)

  • 定義: データ自体から自動的に教師信号（ラベル）を生成して学習する
  • 仕組みの例:
    • BERT: 文章の一部を隠して予測させる（Masked Language Modeling）
    • GPT: 次のトークンを予測させる（Causal Language Modeling）
  • 重要性: 人手でのラベリング不要で、インターネット上の膨大なテキストを学習できる。LLMの事前学習（Pre-training）の基礎。

モデルの適合性とバイアス

過学習 (Overfitting) と未学習 (Underfitting)

問題 状態 原因 対策
過学習
(Overfitting)		 訓練データに過度に適合し、未知データへの汎化性能が低下した状態。訓練誤差は小さいが、テスト誤差が大きい。
  • モデルが複雑すぎる
  • 訓練データが少ない
  • 学習しすぎ（エポック過多）
  • 正則化（L1/L2）
  • ドロップアウト（Dropout）
  • データ拡張（Data Augmentation）
  • 早期停止（Early Stopping）
  • 交差検証（Cross-Validation）
未学習
(Underfitting)		 訓練データにも十分適合できない状態。訓練誤差もテスト誤差も大きい。
  • モデルが単純すぎる
  • 特徴量が不足している
  • 学習不足（エポック不足）
  • より複雑なモデルを選択
  • 特徴量エンジニアリング
  • 学習率・エポック数の調整

バイアス・バリアンス トレードオフ

期待誤差 = バイアス² + バリアンス + 不可避ノイズ
  • 高バイアス（High Bias） = 未学習の状態。モデルが単純すぎて真のパターンを捉えられない。
  • 高バリアンス（High Variance） = 過学習の状態。モデルがデータのノイズまで学習してしまう。
  • トレードオフ: バイアスを下げるとバリアンスが上がる傾向があり、両者を同時に下げることは困難。

データバイアスの種類

重要: データバイアスは、AIモデルが不公平・差別的な予測をする根本原因となります。AIF-C01では「責任あるAI」の観点からも頻出テーマです。
バイアスの種類 説明 具体例
選択バイアス 母集団を代表しないデータを収集してしまう偏り。 医療データで特定の人種・年齢層が少ない
確認バイアス 既存の仮説を支持するデータのみを集め、反証するデータを無視する偏り。 「製品が優れている」という前提でレビューを収集
歴史的バイアス 過去の社会的差別・偏見がデータに反映されている偏り。 過去の採用データで女性の採用率が低い → 採用モデルが女性を低評価
測定バイアス データの収集方法・測定方法・ラベリングが不正確であることによる偏り。 センサーの精度が低いグループのデータが不正確

2-2 AIのユースケース

AIが適切な場合とそうでない場合

AIが適切な場合 AIが不適切な場合
  • パターン認識が必要（画像分類、音声認識）
  • 大量データからの予測（需要予測、不正検知）
  • 繰り返し作業の自動化（文書分類、翻訳）
  • 人間には不可能なスピードでの処理（リアルタイム推薦）
  • 大量かつ高品質な学習データが存在する
  • わずかな誤りが許容される用途
  • 高度な倫理的・道徳的判断が必要な場面
  • 学習データが極端に少ない
  • 明確なルールで解決できる（単純な計算等）
  • 説明責任が厳格に求められる（一部の医療・法律判断）
  • 誤りが重大な結果を招く高リスク環境（核兵器管理等）

2-3 ML開発のライフサイクル

MLOps

MLOps とは

MLOps（Machine Learning Operations）は、機械学習システムの本番環境への展開・運用を効率化するための実践・ツール・組織文化の総称です。ソフトウェア開発の DevOps をML分野に応用した概念です。

  • 目標: 再現性・スケーラビリティ・自動化・継続的モニタリングの実現
  • 主要概念:
    • CI（継続的インテグレーション）: コード・データ・モデルの変更を自動でテスト
    • CD（継続的デリバリー）: 検証済みモデルを自動で本番環境にデプロイ
    • CT（継続的トレーニング）: 新データで定期的にモデルを再学習
    • モデルバージョニング: モデルのバージョン管理と再現性の確保
    • フィーチャーストア（Feature Store）: 特徴量の一元管理・共有・再利用
AWS MLOps ツール 役割
SageMaker Pipelines ML ワークフロー（データ前処理→学習→評価→デプロイ）をコードで定義・自動化するCI/CDパイプライン
SageMaker Model Registry 学習済みモデルのバージョン管理、承認ワークフロー、メタデータ管理
SageMaker Model Monitor 本番モデルのデータドリフト・モデル品質・バイアスをリアルタイムで監視
SageMaker Feature Store 特徴量の保存・共有・再利用。オンラインストア（低レイテンシー推論用）とオフラインストア（学習用）を提供
SageMaker Experiments 学習実験の追跡・比較・管理

問題定義

  • AWS Glue: データカタログ（メタデータ管理）、ETL（抽出・変換・ロード）処理。サーバーレスで動作。
  • Amazon Kinesis: ストリームデータのリアルタイム収集・処理
  • Amazon S3: MLデータレイクの標準ストレージ

データ前処理・特徴量エンジニアリング

ML性能の80%は「データの質」で決まるとも言われる最重要ステップ。

処理 手法
欠損値処理 行の削除、平均値/中央値（数値）・最頻値（カテゴリ）での補完、予測補完
外れ値処理 IQR（四分位範囲）ベース除去、Winsorization（外れ値をパーセンタイル値に置換）
正規化・標準化 Min-Max正規化（0〜1にスケーリング）、Z-score標準化（平均0・標準偏差1）
カテゴリ変数のエンコード One-Hot Encoding（カテゴリ間に順序がない場合）、Label Encoding（順序あり場合）
特徴量選択 相関分析、特徴量重要度（XGBoost等）、主成分分析（PCA）

• SageMaker Data Wrangler: GUIベースのデータ前処理・可視化・変換ツール。300以上の組み込みデータ変換を提供。
• SageMaker Feature Store: 処理済み特徴量を保存・管理・チームで共有。リアルタイムとバッチの両用途に対応。

モニタリングと再学習

SageMaker Model Monitor の監視機能

  • データドリフト（Data Drift）: 本番データの統計的分布が学習時から変化していないかを監視
  • モデル品質（Model Quality）: 予測精度の劣化を監視（Ground Truthとの比較）
  • バイアスドリフト（Bias Drift）: モデルの公平性指標の変化を監視（SageMaker Clarifyと統合）
  • 特徴量重要度のドリフト（Feature Attribution Drift）: SHAP値の変化を監視
再学習のトリガー: 性能劣化の検知、データドリフトの検知、新しい大量データの蓄積、定期スケジュール（例: 週次）などがあります。SageMaker Pipelines と EventBridge を組み合わせて自動化できます。

スクラッチから構築 最大の柔軟性。専門的なMLの知識が必要。TensorFlow、PyTorch等を使用。 SageMaker + カスタムコード（Bring Your Own Container も可）

  • コスト・手間が低い順: Bedrock API > JumpStart > 組み込みアルゴリズム > スクラッチ

分類モデルの評価

分類モデルの評価は「混同行列（Confusion Matrix）」を基にします。

分類モデルの主要評価指標

正解率（Accuracy） = (TP + TN) / (TP + TN + FP + FN)
適合率（Precision） = TP / (TP + FP) ← 「陽性と予測したもののうち、実際に陽性の割合」
再現率（Recall） = TP / (TP + FN) ← 「実際の陽性のうち、正しく検出できた割合」
F1スコア（F1-Score）= 2 × (Precision × Recall) / (Precision + Recall)
AUC-ROC : 閾値に依存しない性能指標（0.5 = ランダム、1.0 = 完全）
指標 重視すべきシナリオ 具体例
適合率（Precision） 偽陽性（False Positive）を減らしたい場合 スパムフィルター（正常メールをスパム判定するのは困る）
再現率（Recall） 偽陰性（False Negative）を減らしたい場合 がん検診（病気を見逃すのは命に関わる）
F1スコア Precision と Recall のバランスが重要な場合 情報検索、不均衡データセットでの評価
AUC-ROC 分類閾値を決める前のモデル全体の識別能力を評価したい場合 信用スコアリング、医療診断モデルの比較

回帰モデルの評価

回帰モデルの主要評価指標

MAE（平均絶対誤差） = (1/n) × Σ|実際値 - 予測値|
MSE（平均二乗誤差） = (1/n) × Σ(実際値 - 予測値)²
RMSE（二乗平均平方根誤差） = √MSE ← 元データと同じ単位で解釈できる
R²（決定係数） = 1 - (Σ(y - ŷ)² / Σ(y - ȳ)²) ← 0〜1、1が完全
指標 外れ値への感度 特徴
MAE 低い（頑健） 各誤差を均等に評価。直感的に理解しやすい。
MSE 高い（敏感） 大きな誤差を二乗するため外れ値が大きく影響する。最適化に使われる。
RMSE 高い（敏感） MSEの平方根で元データと同じ単位になる。最もよく使われる指標。
R² 中程度 モデルの説明力を0〜1で表す。1に近いほど良い。

生成AIモデルの評価

評価指標 用途 説明
BLEU
(Bilingual Evaluation Understudy)		 機械翻訳 参照文との n-gram の一致率を計算。0〜1、高いほど良い。翻訳品質の標準的指標。
ROUGE
(Recall-Oriented Understudy for Gisting Evaluation)		 要約、テキスト生成 参照要約との n-gram 再現率を計算。ROUGE-1（ユニグラム）、ROUGE-L（最長共通部分列）等がある。
BERTScore テキスト生成全般 BERT の文脈埋め込みを使った意味的類似度の評価。表層的な一致ではなく意味を評価。
Perplexity（困惑度） 言語モデル評価 モデルがテキストをどれだけ予測できるかの不確かさ。低いほど良い。
Human Evaluation（人手評価） 全般 人間が「流暢さ」「関連性」「正確性」「有害性の有無」を評価。最も信頼性が高いが高コスト。
試験ポイント: BLEU は機械翻訳の品質評価、ROUGE はテキスト要約の品質評価に使われます。自動評価指標（BLEU/ROUGE）は人手評価と完全には一致しないため、重要なタスクでは人手評価を組み合わせることが推奨されます。

練習問題

以下の問題を解いて、第2章の理解を確認してください。

練習問題　（全6問）

問題 1

顧客の購買履歴データを使って顧客をグループに分類し、それぞれのグループに最適なマーケティングメッセージを送りたい。購買履歴データにはラベル（グループ情報）がない。このユースケースに最も適した機械学習の手法はどれか？

  1. 教師あり学習（Supervised Learning）
  2. 教師なし学習（Unsupervised Learning）
  3. 強化学習（Reinforcement Learning）
  4. 自己教師あり学習（Self-Supervised Learning）
正解：B

ラベル（グループ情報）なしのデータから類似した顧客をグループに分ける作業はクラスタリングです。クラスタリングは教師なし学習の代表的なタスクです。AWSでは SageMaker の組み込みアルゴリズム「K-Means」が使用できます。

A（誤）： 教師あり学習にはラベルデータが必要。

C（誤）： 強化学習はエージェントが環境と相互作用し報酬を最大化するため、用途が異なる。

D（誤）： 自己教師あり学習はデータ自身からラベルを生成する手法で、主にLLMの事前学習に使われる。

問題 2

病院が癌検診のAIモデルを評価しています。「癌を見逃す」リスクを最小限にしたい場合、最も重視すべき評価指標はどれか？

  1. 正解率（Accuracy）
  2. 適合率（Precision）
  3. 再現率（Recall）
  4. AUC-ROC
正解：C

「癌を見逃す」とは偽陰性（False Negative）が発生することを指します。再現率（Recall）= TP / (TP + FN) を最大化することで偽陰性を最小化できます。

A（誤）： 正解率は不均衡データでは指標として不適切。

B（誤）： 適合率は偽陽性を減らしたい場面で重視。

D（誤）： AUC-ROC は総合的指標だが、見逃し最小化を直接制御するのは再現率。

問題 3

あるMLモデルが訓練データでは99%の精度を達成しているが、テストデータでは65%の精度しか出ない。この状態を最も適切に表しているのはどれか？また、この問題への対策として適切なものを一つ選べ。

  1. 未学習（Underfitting）— 対策: より複雑なモデルへの変更
  2. 過学習（Overfitting）— 対策: L1/L2正則化の適用
  3. 未学習（Underfitting）— 対策: 学習率の増加
  4. 過学習（Overfitting）— 対策: より多くのエポックで学習する
正解：B

訓練データで高い精度（99%）を示す一方、テストデータでは大幅に低い（65%）のは「過学習（Overfitting）」の典型的な症状です。正則化（L1/L2）でモデルの複雑さを抑制することで過学習を防げます。

D（誤）： さらに学習を続けると過学習がより深刻になる。

A・C（誤）： 未学習は訓練データでもテストデータでも精度が低い状態。

問題 4

EC サイトで商品ごとの日次売上を翌30日分予測するシステムを構築したい。SageMaker の組み込みアルゴリズムとして最も適切なものはどれか？

  1. XGBoost
  2. K-Means
  3. DeepAR
  4. BlazingText
正解：C

DeepAR は SageMaker の組み込み時系列予測アルゴリズムです。深層学習（RNN）ベースで、複数の関連時系列を同時に学習し、将来の値と予測区間を出力できます。なお、Amazon Forecast も同様のユースケースをカバーします。

A（誤）： XGBoost は構造化データの分類・回帰向けで時系列の順序関係をモデル化しない。

B（誤）： K-Means はクラスタリング（教師なし学習）で時系列予測には使用しない。

D（誤）： BlazingText はテキスト分類（Word2Vec）に特化したアルゴリズム。

問題 5

機械翻訳モデルを評価するために最もよく使われる自動評価指標はどれか？

  1. ROUGE
  2. BLEU
  3. F1スコア
  4. Perplexity（困惑度）
正解：B

BLEU（Bilingual Evaluation Understudy）は、機械翻訳の出力文と参照翻訳の間の n-gram 一致率を計算する標準的な自動評価指標です。

A（誤）： ROUGE はテキスト要約の評価に使われる。

C（誤）： F1スコアは分類タスクの評価指標。

D（誤）： Perplexity は言語モデル自体の不確かさを評価する指標。

問題 6

ML チームが本番環境にデプロイしたモデルを監視しています。最近、入力データの統計的分布が学習時のデータから大きくずれてきていることを検知しました。この現象の名称と、それを自動的に監視するために使用すべき AWS サービスの正しい組み合わせはどれか？

  1. モデルドリフト — SageMaker Experiments
  2. データドリフト — SageMaker Model Monitor
  3. 過学習 — SageMaker Automatic Model Tuning
  4. コンセプトドリフト — SageMaker Data Wrangler
正解：B

入力データの統計的分布が学習時から変化する現象は「データドリフト（Data Drift）」と呼ばれます。Amazon SageMaker Model Monitor が本番エンドポイントへの入力データとベースラインを継続的に比較し、統計的な乖離を検出してアラートを送信します。

A（誤）： SageMaker Experiments は実験管理・追跡ツール。

C（誤）： SageMaker AMT はハイパーパラメータ最適化ツール。

D（誤）： Data Wrangler はデータ前処理ツールで監視機能はない。
