1-1 AWS認定AIプラクティショナー試験の概要

AWS認定試験の全体像

AWSは、クラウドの専門知識を証明する認定資格プログラムを提供しています。認定は難易度に応じて4つのレベルに分類されており、AWS認定AIプラクティショナー (AIF-C01) は最も入門的な ファンデーショナル レベルに属します。

レベル 対象者 代表的な認定資格 推奨経験
ファンデーショナル クラウド入門者・ビジネス担当者 AWS Cloud Practitioner
AWS AI Practitioner		 6ヶ月程度
アソシエイト 実務経験のあるエンジニア Solutions Architect, Developer, SysOps, ML Associate 1年以上
プロフェッショナル 上級エンジニア・アーキテクト Solutions Architect Pro, DevOps Engineer Pro 2年以上
スペシャルティ 特定分野の専門家 ML Specialty, Security, Networking, Database 5年以上（分野別）
AIF-C01はファンデーショナルレベルのため、深いプログラミングやML実装の知識は不要です。AIサービスの概念・ユースケース・倫理的考慮事項を理解することが中心です。Cloud Practitioner (CLF-C02) の取得者はインフラ知識があるため、有利に学習を進められます。

試験形式

  • 多肢選択式 (Single Choice): 4択から1つを選択する形式。問題数の大半を占める。
  • 複数回答式 (Multiple Response): 5択以上から「最も適切な2〜3つ」を選択する形式。部分点はなく、すべて正解で初めて得点になる。

複数回答式の問題は「すべて選択してください」ではなく「最も適切な2つを選んでください」という形式です。問題文に必ず「2つ選択してください」「3つ選択してください」と明記されるので見落とさないようにしましょう。

本試験の特徴

AIF-C01は2024年に新設された比較的新しい試験です。Amazon Bedrock（生成AIのマネージドサービス）や責任あるAIの概念が大きなウェイトを占めており、従来のML試験とは異なる出題傾向があります。生成AIブームを反映した試験構成となっています。

1-2 推奨されるAWSの知識

AWSとは / クラウドコンピューティングの基礎

クラウドコンピューティングの定義

クラウドコンピューティングとは、ITリソース（サーバー、ストレージ、データベース、ネットワーク等）をインターネット経由でオンデマンドに提供し、使用した分だけ料金を支払うモデルです。AWSは世界最大規模のクラウドプロバイダーです。

クラウドの6つの利点（AWS公式）

  • 固定費から変動費へ: 初期投資不要、使った分だけ払う
  • スケールのメリット: AWSの規模により低コストで高品質のインフラ
  • 容量推測が不要: オンデマンドでスケールアップ/ダウン可能
  • 俊敏性と柔軟性: 数分でリソースをプロビジョニング可能
  • データセンター運用コストの削減: AWSに任せて本業に集中
  • 数分でグローバル展開: 世界中のリージョンに即座にデプロイ

クラウドサービスモデル (IaaS / PaaS / SaaS)

モデル 正式名称 管理範囲 AWSの例 AI/MLの例
IaaS Infrastructure as a Service OS・アプリはユーザー管理 EC2, S3, VPC EC2 GPU (P4インスタンス)
PaaS Platform as a Service アプリロジックのみ管理 Elastic Beanstalk, RDS Amazon SageMaker
SaaS Software as a Service 設定・データのみ管理 Amazon WorkMail Amazon Rekognition, Comprehend, Bedrock

試験では「最小限の管理工数でAIを利用したい」シナリオには SaaS型のAIサービス（Rekognition, Comprehend, Bedrock等） が正解になることが多いです。逆に「カスタムモデルを訓練したい」場合は SageMaker（PaaS） が適切です。

AWSグローバルインフラストラクチャ

概念 説明 AI/MLとの関連
リージョン 地理的に分離された場所。現在33以上のリージョンが存在。各リージョンは独立して動作 モデルの訓練・推論をどのリージョンで行うか。Bedrockのモデル提供リージョンが限定される場合がある
アベイラビリティゾーン (AZ) 各リージョンに2つ以上（多くは3つ）の物理的に分離されたデータセンター群。高可用性を実現 SageMakerエンドポイントの高可用性デプロイ、トレーニングジョブの冗長化

ローカルゾーン 主要都市近郊にAWSインフラを拡張した施設。超低レイテンシが必要なワークロード向け リアルタイム推論が要求されるエッジAIアプリケーション

リージョン選択の4つの基準

1. 法令遵守・データ主権

個人情報保護法、GDPR等の規制によりデータを特定の国に保管する必要がある場合。日本企業は「ap-northeast-1 (東京)」が第一候補。

2. レイテンシ（遅延）

ユーザーに最も近いリージョンを選択することで、AIサービスへのアクセス遅延を最小化。

3. サービス提供状況

すべてのAWSサービスがすべてのリージョンで利用可能とは限らない。Bedrockの特定モデルは一部リージョンのみ対応。

4. コスト

リージョンによって同じサービスでも料金が異なる。米国東部 (us-east-1) が最もコストが低いことが多い。

AWS責任共有モデル

AWSとユーザーはセキュリティの責任を分担しています。このモデルを正確に理解することは、試験頻出のテーマです。

AWSの責任
「クラウドのセキュリティ」

  • 物理データセンターのセキュリティ
  • ハードウェア・ファームウェアの保護
  • ハイパーバイザー・仮想化レイヤー
  • マネージドサービスの基盤インフラ
  • グローバルネットワーク設備
  • AWSの従業員のアクセス管理

ユーザーの責任
「クラウド内のセキュリティ」

  • OS・アプリのパッチ適用
  • IAMユーザー・ロール・ポリシー管理
  • データの暗号化（転送中・保存中）
  • セキュリティグループ・ファイアウォール設定
  • 訓練データ・モデルの保護
  • AIの出力に関する品質と倫理管理
Amazon SageMakerの例: AWSが基盤インフラ・MLプラットフォームを管理 ／ ユーザーがモデル・訓練データ・エンドポイント設定を管理
「データの分類・暗号化・IAM設定」は常にユーザー責任です。AIサービスであっても、訓練データの個人情報保護・アクセス権管理はユーザーが行う必要があります。

AWS Identity and Access Management (AWS IAM)

IAMの主要コンポーネント

  • IAMユーザー: 個人や自動化プロセスに対応するエンティティ。アクセスキー・パスワードで認証。
  • IAMグループ: ユーザーの集合。グループに割り当てたポリシーをメンバー全員に適用。
  • IAMロール: EC2・Lambda・SageMakerなどのAWSサービスやフェデレーションユーザーが引き受けるID。パスワードなしで一時的な認証情報を取得。
  • IAMポリシー: 何ができて何ができないかを定義するJSONドキュメント。ユーザー・グループ・ロールにアタッチ。

IAMポリシーの種類

種類 説明 ユースケース
AWS管理ポリシー AWSが提供するスタンドアロンポリシー（例: AmazonS3ReadOnlyAccess） よく使われる権限の素早い付与
カスタマー管理ポリシー ユーザーが独自に作成したスタンドアロンポリシー 細かいアクセス制御が必要な場合
インラインポリシー 特定のユーザー・グループ・ロールに直接埋め込むポリシー そのエンティティ専用の1対1対応ポリシー

AI/MLにおけるIAMの実例

実例

SageMakerトレーニングジョブがS3にアクセスする場合:
SageMakerサービスが直接アクセスキーを持つのではなく、サービスロール（IAMロール）を引き受けることでS3バケット内の訓練データを読み込みます。このロールには「s3:GetObject」権限のみを持つ最小権限のポリシーをアタッチします。

構成: SageMaker Training Job → IAMロール (AmazonSageMakerRole) → IAMポリシー (S3読み取り権限) → S3バケット（訓練データ）

最小権限の原則（Principle of Least Privilege）: 必要最小限の権限のみを付与する。AIサービスに対しても同じ原則を適用し、訓練データや推論結果へのアクセスを限定することがセキュリティのベストプラクティスです。

MFA（多要素認証）: ルートアカウントおよびすべての特権ユーザーには必ずMFAを有効化する。特にBedrockやSageMakerの管理者権限を持つユーザーには必須です。

AWSのサービス料金モデル

従量課金制の3つの原則

  • 使用した分だけ支払う: 初期費用なし、サーバーを止めれば課金も止まる
  • 長期利用でさらに割引: リザーブドインスタンス / Savings Plansで最大72%割引
  • 大量利用で単価が下がる: S3やデータ転送は利用量が増えるほど単価低下
コスト削減手段 説明 AI/MLでの活用
リザーブドインスタンス (RI) 1年または3年の利用を約束してオンデマンドより最大72%割引 常時稼働のSageMakerエンドポイント用GPU EC2
Savings Plans コンピュート使用量を金額でコミット。柔軟性が高い（EC2/Lambda/Fargate対応） ML推論ワークロードの固定費削減
スポットインスタンス 余剰容量を最大90%割引で利用。中断の可能性あり SageMakerのトレーニングジョブ（チェックポイント保存で中断対応）
オンデマンド 予約なし、いつでも利用開始・停止可能。最も高い単価 開発・テスト環境、不定期なバッチ推論

ML学習コスト管理のベストプラクティス

  • AWS Cost Explorer: サービス別・日別のコストを可視化。SageMakerの訓練コストをモニタリング
  • AWS Budgets: 月間予算アラートを設定。SageMaker費用が閾値を超えたら通知
  • AWS Trusted Advisor: コスト最適化の推奨事項を自動提案（未使用リソースの検出など）
  • スポットインスタンス活用: ML訓練ジョブはスポット中断に対応しやすいため、積極的に活用

1-3 対象サービス

AI / 機械学習サービス

Amazon SageMaker

MLの開発・訓練・デプロイを支援するフルマネージドMLプラットフォーム。

  • Studio: ML開発統合IDE
  • Training: 分散訓練ジョブ管理
  • Endpoints: リアルタイム推論
  • Pipelines: MLワークフロー自動化
  • Feature Store: 特徴量管理・共有
  • Model Monitor: データドリフト検知

Amazon Bedrock

複数の基盤モデル（FM）をAPIで利用できるサーバーレスの生成AIサービス。

  • Claude (Anthropic), Titan (Amazon)
  • Llama (Meta), Mistral AI等
  • Knowledge Bases: RAGの構築
  • Agents: 自律的なAIエージェント
  • Guardrails: 安全フィルタリング
  • Fine-tuning (微調整) 対応

Amazon Rekognition

画像・動画に対するビジョンAIサービス。モデル訓練不要。

  • 顔認識・顔比較・顔検索
  • 物体・シーン検出
  • テキスト検出 (OCR)
  • 不適切コンテンツ検出
  • 個人用保護具 (PPE) 検出
  • Custom Labels (カスタムモデル)

Amazon Comprehend

NLPサービス。テキストから洞察を抽出。

  • 感情分析（ポジティブ/ネガティブ等）
  • エンティティ認識（人名・組織・場所等）
  • 言語検出（100以上の言語）
  • キーフレーズ抽出
  • PII（個人識別情報）検出・匿名化
  • Custom Classifier/Entity Recognizer

Amazon Translate

ニューラル機械翻訳サービス。75以上の言語ペアに対応。

  • リアルタイム翻訳 / バッチ翻訳
  • カスタム用語集 (Custom Terminology)
  • アクティブカスタマイズ機能
  • 多言語アプリの構築

Amazon Polly

テキストを自然な音声に変換するTTS (Text-to-Speech) サービス。

  • 60以上の音声・40以上の言語
  • Standard / Neural / Long-Form
  • SSML (音声合成マークアップ) 対応
  • 読み上げアプリ・ナレーション生成

Amazon Lex

会話AIチャットボット構築サービス（AlexaのNLU技術と同じ）。

  • インテント・スロット・フルフィルメント
  • 音声/テキスト入力両対応
  • Lambda連携でバックエンド処理
  • コンタクトセンターへの組み込み

Amazon Transcribe

音声をテキストに変換するSTT (Speech-to-Text) サービス。

  • リアルタイム/バッチ文字起こし
  • 話者識別 (Speaker Diarization)
  • カスタム語彙・カスタムモデル
  • 医療向け (Transcribe Medical)

Amazon Forecast

時系列データの需要予測サービス。ML知識不要でモデルを自動選択。

  • DeepAR+, NPTS, CNN-QR等
  • 在庫管理・需要予測・エネルギー需要
  • 関連時系列データの活用
  • 自動モデル選択 (AutoPredictor)

Amazon Personalize

リアルタイムパーソナライズドレコメンデーションエンジン。

  • ECサイトの商品推薦
  • コンテンツ（映像・記事）推薦
  • ユーザー行動データを自動学習
  • コールドスタート対応

Amazon Textract

ドキュメントからテキスト・構造・フォームデータを抽出するサービス（OCR+構造認識）。

  • PDFスキャン・画像対応
  • フォームのキー・バリュー抽出
  • テーブル構造の抽出
  • Queries機能（特定情報をQAで抽出）

Amazon Kendra

機械学習を活用したインテリジェント企業内検索サービス。

  • 自然言語クエリで直接回答を返す
  • S3・SharePoint・Confluence等に接続
  • セキュリティフィルタリング対応
  • RAGの検索バックエンドとして利用

Amazon Q

生成AIを活用したビジネス・開発向けアシスタントサービス。

  • Q Business: 社内データへの質問応答

ストレージサービス

Amazon S3: MLデータの中央リポジトリ

Amazon S3 (Simple Storage Service) はAWS機械学習ワークフローの中心的なストレージです。SageMakerの訓練データ・モデルアーティファクトの保存先として必ずと言っていいほど登場します。

  • 耐久性: 99.999999999%（11ナイン）の設計耐久性
  • ストレージクラス: Standard / Intelligent-Tiering / Glacier（コストに応じて選択）

  • ライフサイクルポリシー: 古いデータを自動でGlacierに移行してコスト削減
  • イベント通知: 新データ到着時にLambdaを起動して自動訓練パイプライン構築
  • 暗号化: SSE-S3 / SSE-KMS / SSE-C でデータを暗号化保護

問題 1

AWS認定AIプラクティショナー (AIF-C01) 試験に関する説明として、正しいものはどれですか？

  1. この試験はプロフェッショナルレベルに分類され、AWS実務経験5年以上が推奨される。
  2. この試験はファンデーショナルレベルに分類され、合格スコアは1000点満点中720点である。
  3. この試験の試験時間は90分で、問題数は50問である。
  4. この試験では、深層学習モデルの実装コードを記述する問題が出題される。
正解：B

AIF-C01はファンデーショナルレベルに分類され、試験時間は130分・65問（採点対象50問）・合格スコア720点（1000点満点）です。コードの記述は求められず、AIサービスの概念・ユースケース・倫理的考慮事項の理解が中心です。

問題 2

ある企業が、顧客からの問い合わせメールを自動で「感情分析」し、ポジティブ・ネガティブ・ニュートラルに分類するシステムを構築したいと考えています。ML知識がない担当者でも最小限の実装工数で実現できるAWSサービスとして最も適切なものはどれですか？

  1. Amazon SageMaker でカスタム分類モデルを訓練する。
  2. Amazon Rekognition を使用してテキストを分析する。
  3. Amazon Comprehend を使用して感情分析APIを呼び出す。
  4. Amazon Textract でメールのテキストを構造化データとして抽出する。
正解：C

Amazon Comprehend はNLPのマネージドサービスで、感情分析（ポジティブ/ネガティブ/ニュートラル/混在）をAPIの呼び出しだけで実現できます。ML知識やモデル訓練は不要です。Rekognition は画像・動画向け、Textract はドキュメントからのデータ抽出、SageMaker はカスタムMLモデル開発に用います。

問題 3

AWS責任共有モデルに関する説明として、正しいものはどれですか？2つ選択してください。

  1. AWSは物理データセンターのセキュリティとハードウェアの保護に責任を持つ。
  2. Amazon SageMaker を使用する場合、訓練データのアクセス制御はAWSが自動的に行う。
  3. ユーザーは、IAMポリシーの設定やS3バケット内データの暗号化に責任を持つ。
  4. マネージドサービスを使用すると、すべてのセキュリティ責任がAWSに移転する。
  5. ユーザーはAWSのハイパーバイザーのセキュリティパッチ適用を担当する。
正解：A、C

責任共有モデルでは、AWSが「クラウドのセキュリティ」（物理インフラ、ハイパーバイザー等）を担当し、ユーザーが「クラウド内のセキュリティ」（IAM設定、データ暗号化、OSパッチ等）を担当します。

A（正）： AWSは物理データセンターのセキュリティとハードウェアの保護に責任を持ちます。

C（正）： SageMakerを使っても、訓練データへのIAMアクセス制御はユーザーの責任です。

E（誤）： ハイパーバイザーのパッチ適用はAWSの責任です。

問題 4

AIF-C01試験のドメイン構成について、配点が最も高いドメインはどれですか？

  1. ドメイン1: AIと機械学習の基礎（20%）
  2. ドメイン2: 生成AIの基礎（24%）
  3. ドメイン3: 基盤モデルの応用（28%）
  4. ドメイン4: 責任あるAI（14%）
正解：C

ドメイン3「基盤モデルの応用」が28%と最も高い配点を持ちます。Amazon Bedrockを使ったRAG（検索拡張生成）の構築、エージェント、プロンプトエンジニアリング、微調整（Fine-tuning）などが中心トピックです。ドメイン2（24%）と合わせると生成AI関連で52%を占めるため、これらが試験の最重要テーマです。

問題 5

ある企業のデータサイエンティストが、Amazon SageMakerを使用して機械学習モデルを訓練しています。訓練ジョブがAmazon S3に保存された訓練データを読み込めるようにするための、最もセキュリティのベストプラクティスに沿った方法はどれですか？

  1. S3バケットをパブリックアクセスに設定して、SageMakerが直接アクセスできるようにする。
  2. アクセスキーとシークレットキーをSageMakerの環境変数に直接埋め込む。
  3. 必要最小限のS3アクセス権限を持つIAMロールを作成し、SageMakerトレーニングジョブにそのロールを割り当てる。
  4. ルートアカウントの認証情報をSageMakerノートブックに設定する。
正解：C

最小権限の原則に従い、S3への読み取りアクセスのみを許可するIAMロールを作成し、SageMakerトレーニングジョブにアタッチするのが正しいアプローチです。S3のパブリック公開・アクセスキーの直接埋め込み・ルートアカウント使用はいずれも重大なセキュリティリスクです。AWSではアクセスキーよりIAMロールを使用することを強く推奨しています。
