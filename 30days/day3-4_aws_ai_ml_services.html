2️⃣ 完全管理型AIサービスの詳細

⏱️ 学習時間: 約40分

概要図

ユーザーの課題          利用するAWSサービス         結果
─────────────────────────────────────────────────────

1. Amazon Rekognition - 画像・ビデオ分析

👁️Amazon Rekognition

画像およびビデオを分析して、オブジェクト、人物、テキスト、シーンなどを自動認識するサービス。

主な機能：
  • 物体検出 - 画像内のすべてのオブジェクトを検出
  • 顔認識 - 顔の検出、比較、確認
  • 顔分析 - 年齢、性別、感情の推定
  • テキスト認識 - 画像内のテキストを抽出
  • シーン検出 - 背景や環境を分析
  • 不適切コンテンツ検出 - 不適切な画像を判定
ユースケース例：
  • セキュリティカメラの映像分析
  • eコマースの商品分類の自動化
  • 医療画像診断の支援
  • SNS上の不適切コンテンツフィルタリング

2. Amazon Comprehend - テキスト分析

📝Amazon Comprehend

自然言語処理（NLP）を使用してテキストから意味と関係を抽出するサービス。

主な機能：
  • 感情分析 - ポジティブ、ネガティブ、ニュートラル判定
  • エンティティ抽出 - 人名、地名、組織名を認識
  • キーフレーズ抽出 - 重要なフレーズを抽出
  • 言語検出 - テキストの言語を特定
  • トピックモデリング - 文書内のトピックを抽出
  • カスタム分類 - ユーザー定義のカテゴリ分類
ユースケース例：
  • カスタマーレビューの感情分析
  • ソーシャルメディアのモニタリング
  • ニュース記事の自動分類
  • メールのスパム/正常判定

3. Amazon Lex - チャットボット・会話AI

💬Amazon Lex

音声とテキストを理解して応答する会話型インターフェースを構築するサービス。

主な機能：
  • 自動音声認識（ASR）
  • 自然言語理解（NLU）
  • インテント認識 - ユーザーの意図を理解
  • スロット埋め込み - 必要な情報を収集
  • 複数チャネル対応 - Web、SMS、電話
ユースケース例：
  • カスタマーサポートチャットボット
  • 銀行の自動電話応答システム
  • ホテル予約システム
  • FAQボット
Amazon Lex の会話フロー例：

ユーザー: 「ビジネスフライトを予約したいです」
    ↓
Lex [インテント認識]: 「BookFlight」と認識
    ↓
Lex [スロット埋め込み]:
  - 出発地: ユーザーに確認 → 東京
  - 目的地: ユーザーに確認 → 大阪
  - 日程: ユーザーに確認 → 明日
    ↓
Lex [確認]: 「東京から大阪へ明日のフライトをお探しですね」
    ↓
Lex [実行]: バックエンドシステムに予約リクエスト
    ↓
ユーザーへ: 「予約完了しました」

4. Amazon Polly - テキスト音声変換

🔊Amazon Polly

テキストを自然な音声に変換するサービス。複数の言語と音声オプションに対応。

主な機能：
  • 自然な音声合成 - 複数の男性・女性音声
  • 多言語対応 - 25+言語
  • 感情表現 - 話し方のスタイルを調整
  • SSML対応 - 音声マークアップ言語
  • リアルタイム音声ストリーム
ユースケース例：
  • eラーニングコンテンツの音声化
  • アクセシビリティ対応（視覚障害者向け）
  • IVR（自動音声応答）システム
  • ゲームやアプリのナレーション

5. Amazon Translate - 機械翻訳

🌐Amazon Translate

ニューラル機械翻訳を使用して、テキストを複数の言語に翻訳するサービス。

主な機能：
  • 71以上の言語をサポート
  • ニューラル機械翻訳で高精度
  • カスタム用語集 - ドメイン特有の用語に対応
  • 一括翻訳 - 大量の文書を処理
ユースケース例：
  • Webサイトの多言語化
  • グローバル企業のドキュメント翻訳
  • カスタマーサービスの多言語対応
  • 国際的なコンテンツの翻訳

6. Amazon Textract - 文書分析

📄Amazon Textract

スキャンされた文書や画像からテキストと構造化データを自動抽出するサービス。

主な機能：
  • テキスト抽出 - OCR機能
  • テーブル抽出 - 表の構造を認識
  • フォーム分析 - フォームフィールドと値を抽出
  • 手書き認識
ユースケース例：
  • 銀行の書類処理自動化
  • 保険請求の自動処理
  • パスポート・運転免許証の読み取り
  • 請求書の自動処理

3️⃣ 言語・テキスト関連サービス

⏱️ 学習時間: 約15分

テキスト処理サービスの比較

サービス 主な機能 入力 出力
Comprehend 感情分析、エンティティ抽出 テキスト 分析結果
Textract テキスト抽出、テーブル解析 画像/PDF テキスト + 構造
Lex 会話インターフェース 音声/テキスト 応答
Translate 機械翻訳 テキスト 翻訳テキスト

テキスト処理の典型的フロー

PDF/画像 → Textract → 構造化テキスト → Comprehend → 分析結果
                                           ↓
                                      感情スコア
                                      エンティティ
                                      キーフレーズ

または

テキスト → Lex → インテント認識 → Translate → 多言語出力
              ↓
              応答生成 → Polly → 音声出力
💡 使い分けのコツ:
  • スキャン文書がある → Textract でテキスト抽出
  • テキストの意味を分析 → Comprehend 使用
  • 会話型インターフェース → Lex 使用

4️⃣ 視覚・画像関連サービス

⏱️ 学習時間: 約20分

画像処理サービスの詳細

1. Amazon Rekognition - 詳細

Rekognition の機能別用途：

【物体・シーン検出】
  入力: 商品画像 → 出力: "コンピュータ", "デスク", "屋内"
  用途: eコマース商品分類、在庫管理

【顔認識】
  入力: 顔画像 → 出力: 顔の位置、照合結果
  用途: セキュリティ、本人確認

【顔分析】
  入力: 顔画像 → 出力: 年齢範囲、性別、感情（喜び、悲しみなど）
  用途: デモグラフィック分析、感情認識

【テキスト検出】
  入力: 文書画像 → 出力: 抽出されたテキスト
  用途: スキャン文書のテキスト化

【不適切コンテンツ検出】
  入力: 画像 → 出力: 不適切度スコア
  用途: SNSモデレーション、ペアレンタルコントロール

2. Amazon Lookout for Vision - 異常検出

🔍Amazon Lookout for Vision

コンピュータビジョンと機械学習を使用して、画像内の異常（欠陥など）を検出。

用途：
  • 製造業での製品品質検査（欠陥検出）
  • 電子機器のはんだ付け検査
  • 建設現場の安全検査

画像処理サービスの選択フロー

画像に何を検出したいか?

汎用物体検出 → Rekognition
異常・欠陥検出 → Lookout for Vision
テキスト抽出 → Textract
例：自動車製造工場での品質検査

各製品の画像を撮影 → Lookout for Vision で欠陥を検出 → 欠陥があれば検査フローへ → 合格品のみ出荷。 この自動化により、検査時間が50%削減でき、人的ミスも減少。

音声処理サービス

1. Amazon Transcribe - 音声テキスト変換

🎙️Amazon Transcribe

音声ファイルをテキストに自動変換するサービス。自動音声認識（ASR）の完全管理版。

主な機能：
  • 自動音声認識（ASR）
  • 複数言語対応
  • ボキャブラリー カスタマイズ
  • 話者分離 - 異なる人の発話を識別
  • リアルタイムストリーミング対応
ユースケース例：
  • 会議録音の自動テキスト化
  • カスタマーコール録音の分析
  • ポッドキャストの字幕生成
  • 医療診察記録の自動文字化

2. Amazon Polly - テキスト音声変換（再掲）

前述の完全管理型AIサービスセクションで詳述

3. Amazon Connect Contact Lens - 通話分析

📞Contact Lens for Amazon Connect

コールセンターの通話を分析して、ビジネスインテリジェンスを抽出するサービス。

分析内容：
  • 感情分析 - カスタマー満足度
  • 通話内容の自動要約
  • キーフレーズの抽出
  • 規制順守の監視

音声処理の典型的フロー

【リアルタイム通話分析】
通話 → Transcribe → テキスト → Comprehend → 感情分析
            ↓              ↓
         字幕         要約
         
【オフライン処理】
録音ファイル → Transcribe → テキスト → Comprehend → 分析
                                        ↓
                                    データベース保存
                                        ↓
                                  レポート生成
💡 Transcribe と Lex の違い:
  • Transcribe: 音声 → テキスト（変換のみ）
  • Lex: 会話 → インテント認識 → 応答（対話型）

6️⃣ 予測・分析関連サービス

⏱️ 学習時間: 約20分

予測とアナリティクスのサービス群

1. Amazon Forecast - 時系列予測

📈Amazon Forecast

機械学習を使用して、時系列データから将来の値を予測するサービス。

主な機能：
  • 自動アルゴリズム選択
  • 複数の予測アルゴリズム（ARIMA、Prophet、ディープラーニング）
  • 精度指標の自動評価
  • 複数の時系列データ同時処理
ユースケース例：
  • 需要予測 - 在庫最適化
  • 売上予測 - 経営計画
  • ウェブトラフィック予測
  • 電力消費予測
例：オンラインショップの在庫管理

過去12ヶ月の売上データ → Forecast → 30日先の需要予測 → 自動発注 → 在庫切れと過剰在庫を防止

2. Amazon Lookout for Metrics - 異常検知

⚠️Amazon Lookout for Metrics

ビジネスメトリクスの異常を自動検出するサービス。異常の根本原因分析も提供。

用途：
  • 売上の異常下落を即座に検出
  • 顧客チャーン率の異常
  • Webサイトのアクセス異常
  • アプリのクラッシュレート異常

3. Amazon Lookout for Equipment - 予測的メンテナンス

🔧Amazon Lookout for Equipment

機械やセンサーデータから故障を予測するサービス。

用途：
  • 工場機械の故障予測
  • 風力発電機のメンテナンス計画
  • ポンプやコンプレッサーの障害検出

予測サービスの比較

予測対象               → 使用するサービス
─────────────────────────────────────
未来の売上、需要        → Amazon Forecast
メトリクスの異常        → Lookout for Metrics
機械の故障リスク        → Lookout for Equipment

4. Amazon Fraud Detector - 不正検出

🛡️Amazon Fraud Detector

機械学習を使用して、オンライン不正取引やアカウント不正を検出。

検出対象：
  • クレジットカード不正取引
  • アカウント乗っ取り
  • ボット攻撃
  • 新規アカウント悪用

5. Amazon Personalize - 推奨エンジン

⭐Amazon Personalize

ユーザーの行動からパーソナライズされた推奨を提供するサービス。

推奨タイプ：
  • 商品推奨 - eコマース
  • 記事推奨 - ニュースサイト
  • 動画推奨 - ビデオストリーミング
  • ユーザー類似性検出

7️⃣ Amazon SageMaker - MLプラットフォーム

SageMaker vs 完全管理型AIサービス

側面 完全管理型AIサービス SageMaker
使いやすさ 非常に簡単（API呼び出し） 技術知識が必要
カスタマイズ 制限あり 完全にカスタマイズ可能
学習曲線 短い 長い
時間 数分で本番化 数日～数週間
コスト 使用量ベース リソース使用量ベース
適用例 汎用タスク カスタムモデルが必要
💡 使い分けの基本:
  • 完全管理型AIで対応できる → それを使う（迅速・簡単）
  • 特別なカスタマイズが必要 → SageMaker を使う

AWS AI/ML サービス使用時のベストプラクティス

1. サービス選択

  • ✓ 完全管理型AIサービスから始める（迅速・低コスト）
  • ✓ 複雑さが必要なときのみSageMakerへ移行
  • ✓ 複数サービスの組み合わせを考慮

2. データ管理

  • ✓ データ品質を最優先に考える
  • ✓ Amazon S3 をデータストレージとして使用
  • ✓ データラベリングは Ground Truth で実施
  • ✓ IAM で厳格なアクセス制御

3. コスト最適化

  • ✓ バッチ処理が可能ならバッチ Transform を使用
  • ✓ スポットインスタンスで訓練コスト削減
  • ✓ 不要なエンドポイントは削除
  • ✓ 適切なインスタンスサイズを選択

4. セキュリティ

  • ✓ VPC エンドポイントを使用してプライベート通信
  • ✓ KMS で保存時の暗号化
  • ✓ SSL/TLS で転送中の暗号化
  • ✓ AWS CloudTrail でログ記録

5. モニタリング

  • ✓ CloudWatch でメトリクス監視
  • ✓ SageMaker Model Monitor でデータドリフト検出
  • ✓ Model Explainability で予測の透明性確保
  • ✓ 定期的なモデル再訓練を計画

6. スケーリング

  • ✓ 予測トラフィック増加を見込んで自動スケーリング設定
  • ✓ マルチモデルエンドポイントで複数モデル共存
  • ✓ バッチ処理で大量データ効率的処理
💡 最も重要なベストプラクティス:
