1. AWSセキュリティの基礎

学習時間の目安: 30分
このセクションの目標: AWSクラウド全体に共通するセキュリティの基盤となるサービス群（IAM・KMS・VPC・CloudTrail・Config）を理解し、AIシステムのセキュリティ設計の土台を固める。

1.1 IAM（Identity and Access Management）

IAMは「誰が」「何に対して」「何をできるか」を定義するAWSの中心的セキュリティサービスです。AIシステムにおいては、SageMaker・Bedrock・S3などへのアクセスを適切に制御するために不可欠です。

IAMユーザー

人間の操作者に割り当てる永続的なアイデンティティ。アクセスキーやパスワードで認証。

注意: ルートアカウントの日常使用は厳禁。MFA必須。

IAMロール

AWSサービスや一時的なユーザーに付与する権限セット。SageMakerがS3に書き込む際などに利用。

重要: AIサービスには必ずロールを使う。

IAMポリシー

何を許可/拒否するかを定義するJSONドキュメント。ユーザー・グループ・ロールにアタッチ。

種類: AWS管理ポリシー、カスタマー管理ポリシー、インラインポリシー

最小権限の原則（Principle of Least Privilege）

各ユーザー・サービス・ロールには、タスクを実行するために
必要最小限の権限のみを付与する

SageMaker実行ロール例: S3特定バケット読み取り + SageMaker実行のみ
NG例: AdministratorAccess を SageMaker ジョブに付与
試験頻出: IAMロールを使う場面 — EC2からS3へアクセス、Lambda関数がDynamoDBを操作、SageMakerジョブがS3からデータを読み込む。これらはすべてIAMロールで権限を付与する。アクセスキーをコードにハードコードしてはいけない。

AWS KMS（Key Management Service）の重要ポイント:
  • CMK（Customer Managed Key）: 顧客が管理する暗号化キー。キーポリシーで使用権限を細かく制御できる。
  • AWS管理キー: AWSが自動管理。特定サービス（S3・SageMaker等）専用。
  • キーローテーション: CMKは年次自動ローテーションを有効化できる。
  • CloudTrail統合: KMSへのすべての呼び出し（暗号化・復号）がCloudTrailに記録される。

1.4 AWS CloudTrail

CloudTrailは、AWSアカウント内のすべてのAPI呼び出し（誰が・いつ・何を・どこから操作したか）を記録する監査サービスです。

管理イベント

コントロールプレーン操作の記録

IAMポリシーの変更・S3バケットの作成・SageMakerモデルのデプロイなど、AWSリソースの設定変更をすべて記録。デフォルトで有効。

データイベント

データ操作の詳細記録

S3のオブジェクト読み書き・DynamoDBのGetItem/PutItem・Lambda関数の実行など。別途有効化が必要（コスト増）。

CloudTrail Insights

異常なAPI活動の自動検出

通常と異なるAPIコール数・エラーレートを機械学習で検出し、アラートを発報。不正アクセスや設定ミスを早期発見。

統合

他サービスとの連携

ログはS3に保存（長期保管）。CloudWatch Logsに転送してリアルタイム監視。Athenaでログを直接SQLクエリ分析。

2. AIシステムのセキュリティ

学習時間の目安: 30分
このセクションの目標: SageMakerやBedrockなどのAIサービス固有のセキュリティ機能を理解し、モデルの知的財産保護・推論エンドポイントの保護・訓練データのアクセス制御を習得する。

2.1 SageMakerセキュリティ

ネットワーク隔離

  • VPCモード: SageMakerのトレーニングジョブ・エンドポイントをVPC内に配置。インターネットへの露出を排除。
  • ネットワーク分離: トレーニングコンテナがインターネットにアクセスできないよう完全隔離（enable_network_isolation=True）。
  • VPCエンドポイント: S3・ECRへのアクセスをインターネット経由ではなくVPC内で完結させる。

エンドポイントセキュリティ

  • HTTPS強制: SageMakerエンドポイントはデフォルトでHTTPS（TLS）のみ受け付ける。
  • 認証: AWS SigV4署名によるリクエスト認証。IAMポリシーで呼び出し権限を制御。
  • プライベートエンドポイント: Interface型VPCエンドポイントでVPC外部からのアクセスを遮断。

2.2 モデルの保護（知的財産・アーティファクト暗号化）

モデルアーティファクトとは: 訓練済みモデルの重みパラメータや構成ファイルのこと。数週間・数ヶ月の計算コストと膨大なデータで生成された企業の重要な知的財産です。

保護する理由: モデルが盗まれると、他社が同等の競合サービスを構築できてしまう。また、モデルを解析することでトレーニングデータの個人情報が漏洩するリスク（モデル逆転攻撃）もある。

3. データプライバシーと規制対応

学習時間の目安: 35分
このセクションの目標: グローバルに展開するAIシステムが遵守すべき主要な個人情報保護法規（GDPR・CCPA・HIPAA）の要件を理解し、AWSのサービスを使ってどのようにコンプライアンスを実現するかを学ぶ。

3.1 GDPR（一般データ保護規則）

GDPRとは: 2018年に施行されたEU（欧州連合）の包括的なデータ保護法。EU市民の個人データを処理するすべての組織に適用される（EU域外の企業も対象）。違反した場合、年間売上の4%または2,000万ユーロの高い方の制裁金が課される。
GDPR要件 内容 AIシステムへの影響 AWSでの対応
データ主体の権利 アクセス権・修正権・削除権（忘れられる権利）・データポータビリティ権 ユーザーデータを訓練に使った場合、削除要求への対応が必要 S3 + DynamoDB でデータ識別・削除パイプライン構築
処理の根拠 同意・契約履行・正当な利益など6つの根拠が必要 AIによるプロファイリングには明示的同意が必要 Cognitoで同意管理
プライバシーバイデザイン システム設計段階からプライバシー保護を組み込む データ最小化・仮名化・暗号化をデフォルトに KMS暗号化・Macie（PII検出）
自動意思決定への権利 純粋に自動化された意思決定に対して人間のレビューを要求できる AIによる採用・与信・保険審査での人間レビュー義務 SageMaker Clarify + 人間レビューワークフロー
データ侵害通知 侵害発覚から72時間以内に当局へ通知 AIシステムへの不正アクセスも対象 GuardDuty + Security Hub + SNS自動通知

3.2 CCPA（カリフォルニア州消費者プライバシー法）

CCPAの主な権利

  • 知る権利: どのような個人情報を収集・使用しているかを知る権利
  • 削除権: 収集した個人情報の削除を要求できる
  • オプトアウト権: 個人情報の販売を拒否できる
  • 差別されない権利: 権利行使を理由とした差別的扱いの禁止

GDPRとの主な違い

  • 適用対象: カリフォルニア州（GDPRはEU全域）
  • オプトイン vs オプトアウト: CCPAはオプトアウト方式（GDPRは事前同意=オプトイン）
  • 罰則: 最大750ドル/件（GDPRの方が高額）
  • B2Bデータ: CCPAは消費者向けのみ（GDPRは幅広い）

3.3 HIPAA（医療情報の取り扱い）

HIPAA違反のリスク: 医療情報（PHI）を適切に保護しないと、最大190万ドル/年の罰金。刑事罰として最長10年の禁固刑も。医療AI開発では最も厳格なコンプライアンスが求められる。

PHI（Protected Health Information）とは

以下の18種類の識別子を含む医療情報はすべてPHIとして保護が必要です:

  • 氏名・住所・生年月日
  • 社会保障番号・医療保険番号
  • 電話番号・メールアドレス
  • MRI/X線画像などの医療画像
  • 診断記録・処方記録
  • IPアドレス・デバイスID
  • 生体認証情報（指紋・網膜）
  • 口座番号・クレジットカード番号
  • URLやWeb URL
  • その他直接・間接的に個人を特定できる情報
HIPAAセーフガード 要件 AWSサービスによる実現
技術的セーフガード 暗号化・アクセス制御・監査ログ・転送時暗号化 KMS・IAM・CloudTrail・ACM（TLS証明書）
物理的セーフガード 施設アクセス制限・ワークステーション保護 AWSデータセンターのISO認証・SOC2対応で提供
管理的セーフガード セキュリティポリシー・スタッフトレーニング・BAA（業務提携契約） AWS BAA（Business Associate Agreement）の締結
AWS HIPAA対応サービス: AWSはHIPAAのBAA（業務提携契約）を締結できるHIPAA適格サービスを提供。主な適格サービス: Amazon S3・EC2・RDS・SageMaker・Comprehend Medical・Transcribe Medical・HealthLake。Bedrockも一部モデルでHIPAA適格。使用前に必ずAWSとBAA締結が必要。

3.4 AWS GovCloud

AWS GovCloudとは: 米国政府機関・防衛産業・国家安全保障に関連する組織向けの専用AWSリージョン（us-gov-west-1・us-gov-east-1）。物理的に米国本土にあり、米国市民のみがアクセス可能。FedRAMP・ITAR・EAR等の厳格なコンプライアンス要件に対応。
規格・法律 対象 GovCloudでの対応
FedRAMP High 連邦政府機関のクラウドサービス GovCloudはFedRAMP High認可済み
ITAR（国際武器取引規則） 防衛・軍事関連の輸出管理 GovCloud内でITAR管理データを処理可能
CJIS（刑事司法情報） FBI等の刑事司法情報 法執行機関のAIシステム展開に対応
DoD CC SRG 米国国防総省のクラウド要件 IL2/IL4/IL5レベルのデータを処理可能

3.5 データ主権とデータレジデンシー

データ主権（Data Sovereignty）とデータレジデンシー（Data Residency）

データ主権: データがどの国の法律に管轄されるかの原則。多くの国・EUでは「自国民のデータは自国法で守られるべき」という考え方を採用。

データレジデンシー: データを特定の地理的場所（国・地域）内に保存・処理することの要件。


AWSでの実現方法:

  • リージョン選択: 東京リージョン（ap-northeast-1）に限定することで日本国内にデータを保持。
  • AWS Organizations + SCP（サービス制御ポリシー）: 特定リージョン以外へのリソース作成を組織全体で禁止。
  • AWS Control Tower: 複数アカウントのガバナンスを一元管理。データレジデンシーポリシーを自動適用。
  • S3オブジェクトレプリケーションの制限: クロスリージョンレプリケーションを無効化し、データが国外に出ないよう設定。
試験頻出: 「EU顧客データをEU域内に保存しなければならない」→ AWSフランクフルト（eu-central-1）またはパリ（eu-west-3）リージョンを指定。SCPでそれ以外のリージョンへのデータ移動を禁止。これが「データレジデンシー」要件への対応。

4. 説明可能性とモデル解釈

学習時間の目安: 30分
このセクションの目標: 「なぜこの予測をしたのか」を説明できるAI（XAI: Explainable AI）の技術と、SageMaker Clarify・SHAP・LIMEなどのツールを理解する。規制上の要件としても重要。

4.1 SageMaker Clarify: バイアス検出と特徴量重要度

SageMaker Clarifyとは: 機械学習モデルの公平性（バイアス）の評価と、モデルの予測根拠の説明を提供するAWSサービス。トレーニング前のデータバイアス検出と、トレーニング後のモデルバイアス検出の両方に対応。

データバイアス指標（訓練前）

  • CI（クラス不均衡）: データセット内のクラスラベルの不均衡度。1に近いほどバイアスが高い。
  • DPL（Difference in Positive Proportions in Labels）: 異なるグループ間でのポジティブラベル割合の差。
  • CDDL（Conditional Demographic Disparity in Labels）: 条件付き人口統計格差。

モデルバイアス指標（訓練後）

  • DPPL（Difference in Positive Proportions in Predicted Labels）: グループ間の予測ポジティブ率の差。
  • DI（Disparate Impact）: 不均衡な影響。1.0に近いほど公平。
  • AD（Accuracy Difference）: グループ間の精度の差。小さいほど公平。
実際のバイアス検出例: 採用AIの公平性評価

シナリオ: 履歴書のAI選考システムを構築。過去の採用データで訓練。

SageMaker Clarifyで発見:
・男性応募者の採用率: 45%、女性応募者の採用率: 28%
・DI = 28 / 45 = 0.62（1.0未満 → 不公平）
・原因: 過去の採用データ自体に性別バイアスが存在（歴史的偏り）

対策: 保護属性（性別・人種）を除外してモデル再訓練 + Clarifyで定期的にバイアス監視

4.2 SHAP値とLIMEによるモデル解釈

手法 フルネーム 仕組み 特徴・使いどころ
SHAP SHapley Additive exPlanations ゲーム理論のShapley値を使い、各特徴量が予測に与えた貢献度を計算 グローバル（全体傾向）とローカル（個別予測）の両方を説明可能。数学的に厳密。SageMaker Clarifyに統合済み。
LIME Local Interpretable Model-agnostic Explanations 予測したい入力の近傍に擾乱データを生成し、局所的に単純な線形モデルで近似 ローカル説明に特化。どんなモデルにも適用可能（モデル非依存）。計算が速い。
特徴量重要度
（Feature Importance）		 — 決定木系モデル（XGBoost等）が学習時に計算する各特徴量の分割への貢献度 グローバルな重要度のみ。計算コストが低い。XGBoostに内蔵。
Partial Dependence Plot PDP 特定の特徴量の値を変化させたとき予測がどう変わるかを可視化 特徴量と予測の関係性を直感的に理解。非線形な依存関係も検出。 
SHAP値の解釈例（ローン審査モデル）:

顧客Aの融資申請: モデルの予測スコア = 0.72（承認）

SHAP値の内訳（各特徴量の貢献）:
  年収 500万円         → +0.25 （承認方向に貢献）
  勤続年数 10年        → +0.18 （承認方向に貢献）
  既存ローン残高 0万円 → +0.15 （承認方向に貢献）
  年齢 28歳            → -0.08 （拒否方向に貢献）
  借入回数 1回         → +0.05 （承認方向に貢献）
  ベースライン（平均） = 0.17

  合計: 0.17 + 0.25 + 0.18 + 0.15 - 0.08 + 0.05 = 0.72 ✓

→ 「年収と勤続年数が承認の主な理由です」と顧客に説明可能

4.3 モデルカード

モデルカード（Model Card）とは

モデルの透明性と説明責任を確保するためのドキュメント。Googleが2018年に提唱し、現在はAI業界標準となりつつある。


モデルカードに含まれる内容:

  • モデルの概要: 目的・アーキテクチャ・バージョン情報
  • 訓練データ: データソース・サイズ・収集方法・前処理内容
  • 評価結果: 精度指標（サブグループ別も含む）
  • 倫理的考慮事項: 既知のバイアス・制限事項・リスク
  • 使用上の注意: 意図した用途・意図しない用途・外挿リスク
  • 保守情報: モデルオーナー・更新スケジュール・問い合わせ先

SageMaker Model Cards: AWSはSageMaker上でモデルカードを作成・管理・共有する機能を提供。PDF/JSONエクスポートに対応。

6.3 重要キーワード一覧

セキュリティ基礎

  • 最小権限の原則: 必要最小限のIAM権限のみ付与
  • 責任共有モデル: AWSがインフラを、顧客がデータ・アプリを保護
  • 多層防御（Defense in Depth）: 複数のセキュリティレイヤーを組み合わせる
  • ゼロトラスト: 「信頼しない、常に検証」の原則
  • KMS CMK: 顧客管理キーでの暗号化

規制・コンプライアンス

  • GDPR: EU個人情報保護法。データ主体の権利・忘れられる権利
  • CCPA: カリフォルニア州プライバシー法
  • HIPAA/PHI: 医療情報保護法・保護医療情報
  • FedRAMP: 米国連邦政府向けクラウド認可
  • データレジデンシー: データを特定の地理的場所に保管する要件

AI特有のセキュリティ

  • モデル逆転攻撃: モデルからトレーニングデータを推測する攻撃
  • モデル抽出攻撃: APIを通じてモデルを複製する攻撃
  • 敵対的サンプル: モデルを誤判定させるよう細工された入力
  • データポイズニング: 訓練データに悪意ある入力を混入させる攻撃
  • プロンプトインジェクション: 生成AIの指示を乗っ取る攻撃

演習問題（AIF-C01 スタイル）

問1. 金融機関がAWSでAI与信審査システムを構築しています。SageMakerのトレーニングジョブは顧客の財務データ（PII含む）をS3バケットから読み込みます。トレーニングジョブのデータアクセスに関して、最小権限の原則に従った最適な設定はどれですか？

A) SageMaker実行ロールにAmazonS3FullAccessポリシーをアタッチする
B) SageMaker実行ロールに対象S3バケットの特定プレフィックスへのs3:GetObjectのみを許可するカスタムIAMポリシーを作成しアタッチする ★正解★
C) AdministratorAccessポリシーをアタッチして確実にアクセスできるようにする
D) アクセスキーをトレーニングスクリプト内にハードコードする

解説: 最小権限の原則は「タスクに必要な権限のみ付与」。S3FullAccessは不要な書き込み・削除権限も含む。AdministratorAccessはすべてのAWSリソースへのフルアクセスで絶対NG。アクセスキーのハードコードはセキュリティリスクが高くNG。特定バケット・特定操作のみ許可するカスタムポリシーが正解。

問2. HIPAAに準拠した医療AIシステムをAWSで構築しています。患者の検査画像（PHI含む）をS3に保存する際のセキュリティ要件として正しいものはどれですか？（2つ選択）

A) S3バケットのパブリックアクセスをブロックする ★正解★
B) S3のSSE-KMSを有効化し、医療データ専用のCMKで暗号化する ★正解★
C) HIPAAはパブリッククラウドへの保存を禁止しているためオンプレミスに保存する
D) 暗号化より検索速度が重要なため暗号化は不要

解説: HIPAAはAWSのようなBAA締結済みのクラウドプロバイダーへのPHI保存を認めている（Cは誤り）。PHIはアクセス制御（パブリックアクセスブロック）と暗号化（SSE-KMS）の両方が技術的セーフガードとして必要。

問3. 採用選考AIシステムを本番運用中に、女性応募者の合格率が男性より著しく低いという報告を受けました。このバイアス問題を調査・是正するために最も適したAWSサービスはどれですか？

A) Amazon CloudWatch（精度メトリクス監視）
B) AWS CloudTrail（API呼び出し監査）
C) Amazon SageMaker Clarify ★正解★
D) AWS Config（設定コンプライアンス評価）

解説: SageMaker Clarifyはモデルのバイアス検出（DI・DPPL等の指標算出）と、SHAP値による特徴量重要度の説明を提供する。性別バイアスのような公平性問題の調査・是正に直接対応するサービスはClarify。CloudWatchはパフォーマンス監視、CloudTrailはAPI操作ログ、ConfigはAWSリソース設定評価。

問4. EU顧客向けのAIレコメンドサービスを提供する日本企業が、GDPR対応としてEU顧客データをEU域内にのみ保管することを要件としています。AWSでこの要件を実現するための最も確実な方法はどれですか？

A) 開発者に口頭でEUリージョン以外を使わないよう指示する
B) AWS Organizations のSCP（サービス制御ポリシー）でEUリージョン（eu-west-1等）以外へのリソース作成を組織全体で拒否する ★正解★
C) EUリージョンでS3バケットを作成すればクロスリージョンレプリケーションは自動的に無効になる
D) AWS GovCloudを使用する

解説: データレジデンシー要件はSCPで組織全体のリソース作成リージョンを強制的に制限することで確実に実現できる。口頭の指示（A）は技術的担保がなく不確実。クロスリージョンレプリケーションは自動ではなく設定が必要（C）。GovCloudは米国政府向けで今回の要件（D）と不一致。

問5. 本番稼働中のSageMakerエンドポイントで、先月から今月にかけて精度が大幅に低下していることが分かりました。入力データの分布が変化した可能性があります。このデータドリフトを継続的に自動検出するために使うべきAWSサービスはどれですか？

A) AWS CloudTrail
B) AWS Config
C) Amazon SageMaker Model Monitor ★正解★
D) Amazon GuardDuty

解説: SageMaker Model Monitorは本番エンドポイントの入力データ分布・モデル品質・バイアス指標・説明可能性を継続的に監視し、ドリフト（変化）を検出してCloudWatchアラームを発報する。CloudTrailはAPI操作の記録、ConfigはAWSリソース設定の評価、GuardDutyはセキュリティ脅威検知のサービス。

問6. 新しいAIシステムへの不正アクセスを調査するため、過去30日間に特定のS3バケット内の訓練データに誰がアクセスしたかを確認する必要があります。どのサービスで確認できますか？

A) Amazon CloudWatch Logs
B) AWS CloudTrail（データイベントを有効化済み） ★正解★
C) SageMaker Model Monitor
D) Amazon Inspector

解説: S3オブジェクトへのアクセス（GetObject等）を記録するにはCloudTrailのデータイベントを有効化する必要がある。デフォルトでは管理イベント（S3バケット作成等）のみ記録される。CloudWatchはアプリログ、Model Monitorはモデルドリフト、InspectorはEC2/コンテナの脆弱性スキャン。
