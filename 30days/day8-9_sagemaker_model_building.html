1️⃣ SageMakerの役割とサービス概要

⏱️ 学習時間: 約20分

Amazon SageMakerは、AI/MLモデルを構築、学習、デプロイ、運用するためのフルマネージドプラットフォームです。SageMakerを使うことで、インフラ管理の負荷を減らし、迅速にモデルを実行できます。

SageMakerが提供する価値

  • データ準備、学習、検証、推論を1つの環境で統合
  • スケーラブルな分散学習と自動スケーリング
  • 組み込みアルゴリズムとフレームワークサポート
  • セキュアな実行環境とIAMによるアクセス制御

💡 重要

SageMakerは「モデル構築のライフサイクル全体」をサポートします。実験の反復、モデルの追跡、デプロイの自動化までを一貫して管理できます。

2️⃣ SageMaker Studioの機能

⏱️ 学習時間: 約20分

Studioの主要機能

  • ノートブック、ターミナル、データプロファイリングの一元管理
  • Data Wranglerによる前処理の可視化
  • Experiment Trackerによる実験管理
  • Model Monitorによるモデル品質監視

Studioで行う典型的なタスク

  • データの読み込みと可視化
  • 前処理ワークフローの構築
  • 学習ジョブの実行とハイパーパラメータ比較
  • モデルのデプロイとエンドポイント管理

3️⃣ データ準備から学習ジョブまでの流れ

⏱️ 学習時間: 約30分

モデル構築ワークフロー

  1. データをS3に保存し、アクセス権限を設定する
  2. SageMaker Studioでノートブックを作成する
  3. Data Wranglerやスクリプトで前処理を実行する
  4. Estimatorで学習ジョブを定義し、学習を開始する
  5. 学習結果を評価し、モデルを保存する

基本的な学習ジョブの構成

  • トレーニングデータのS3パス
  • 学習アルゴリズムまたはコンテナ
  • インスタンスタイプとインスタンス数
  • ハイパーパラメータ・入力/出力設定
例: 画像認識ならGPUインスタンス、表形式データならCPUインスタンスを使うなど、データタイプに応じてリソースを選択します。

4️⃣ SageMakerの学習と推論の主要コンポーネント

⏱️ 学習時間: 約30分

主要コンポーネント

コンポーネント 説明
Notebook Instance/Studio データ分析とモデル構築のインタラクティブな環境
Training Job 学習アルゴリズムを実行するマネージドなジョブ
Model 学習済みアーティファクトを管理するオブジェクト
Endpoint リアルタイム推論用のホスティング環境
Batch Transform バッチ推論を実行するためのジョブ

Training Jobの実行フロー

  1. 学習コンテナを起動
  2. トレーニングデータをダウンロード
  3. 学習処理を実行
  4. モデルアーティファクトをS3に保存
  5. 学習メトリクスをCloudWatchに送信

5️⃣ 組み込みアルゴリズムとカスタムコンテナ

⏱️ 学習時間: 約20分

組み込みアルゴリズムのメリット

  • 事前に最適化されたコンテナを利用できる
  • データ入力形式が標準化されている
  • 初学者でも簡単に学習ジョブを開始できる

代表的な組み込みアルゴリズム

  • XGBoost: 回帰・分類に強い
  • Linear Learner: 線形モデル
  • Random Cut Forest: 異常検知
  • BlazingText: テキスト分類・埋め込み

カスタムコンテナの利用シーン

  • 独自ライブラリを使う必要がある場合
  • 特殊な前処理やモデル定義を持つ場合
  • 既存のTensorFlow/PyTorchコードをそのまま動かしたい場合
from sagemaker.sklearn.estimator import SKLearn

script_path = 'train.py'
sklearn = SKLearn(
    entry_point=script_path,
    role=role,
    instance_type='ml.m5.large',
    framework_version='1.0-1',
    py_version='py3'
)
sklearn.fit({'train': train_input})

6️⃣ ハイパーパラメータチューニングと自動モデル選択

⏱️ 学習時間: 約20分

ハイパーパラメータチューニングの目的

学習アルゴリズムの設定値を自動的に最適化することで、モデル性能を引き上げることができます。これにより、手動で試行錯誤する時間を削減できます。

SageMakerのチューニング機能

  • Grid Search / Random Search / Bayesian Optimization
  • 複数の候補を同時に実行
  • 最適化指標に応じた早期停止
from sagemaker.tuner import HyperparameterTuner, IntegerParameter, CategoricalParameter

xgb = sagemaker.estimator.Estimator(...)

tuner = HyperparameterTuner(
    estimator=xgb,
    objective_metric_name='validation:auc',
    hyperparameter_ranges={
        'eta': CategoricalParameter([0.01, 0.1, 0.2]),
        'max_depth': IntegerParameter(3, 10),
    },
    max_jobs=12,
    max_parallel_jobs=4
)
tuner.fit({'train': train_input, 'validation': validation_input})

自動モデル選択の考え方

  • 複数のアルゴリズムやモデルを比較する
  • 評価指標を統一して比較可能にする
  • 検証セットでの性能と運用コストを合わせて判断する

7️⃣ モデルデプロイと推論パイプライン

⏱️ 学習時間: 約25分

リアルタイム推論とバッチ推論

  • リアルタイム推論: Endpointを使い、低レイテンシでリクエストに応答
  • バッチ推論: Batch Transformで大量データを一括処理

モデルデプロイの基本手順

  1. 学習済みモデルパッケージを作成
  2. EndpointConfigを定義
  3. Endpointを作成
  4. テストリクエストで推論精度を確認
from sagemaker.model import Model

model = Model(
    image_uri=image_uri,
    model_data=model_data,
    role=role,
    sagemaker_session=sagemaker_session
)
predictor = model.deploy(
    initial_instance_count=1,
    instance_type='ml.t2.medium'
)

result = predictor.predict(payload)
print(result)

推論パイプラインの拡張

  • 前処理用のLambdaを組み合わせる
  • Step Functionsで学習・デプロイ・監視を連携
  • モデルバージョン管理とA/Bテストを行う

8️⃣ 運用・監視・コスト管理

⏱️ 学習時間: 約20分

運用で重要なポイント

  • Endpointの可用性とスケールを監視する
  • 予測精度の劣化をモデルモニタで検出する
  • データドリフトや概念ドリフトを追跡する

コスト管理のポイント

  • 不要なNotebook InstanceやEndpointは停止・削除する
  • スポットインスタンスやバッチトレーニングでコスト削減
  • 推論負荷に応じてAutoscalingを設定する
💡 注意: SageMakerは便利だが使いっぱなしにするとコストが膨らみがちです。利用状況に応じたリソース管理と自動停止を組み込みましょう。

9️⃣ 演習: SageMakerでのモデル構築ワークフロー

⏱️ 学習時間: 約20分

演習シナリオ

表形式データを使って分類モデルを構築し、SageMaker上で学習とデプロイを実行します。

ステップ1: S3にデータを準備

  • トレーニングデータと検証データをS3にアップロード
  • バケットのアクセス権限を確認

ステップ2: 学習スクリプトを作成

  • データ読み込み、前処理、モデル訓練、保存処理を記述
  • ハイパーパラメータは引数で取り込む

ステップ3: 学習ジョブを実行

  • Estimatorを使ってジョブを定義
  • ジョブの進行状況とCloudWatchログを確認

ステップ4: 推論と評価

  • エンドポイントを作成し、サンプル入力で推論
  • 混同行列やAUCなどの評価指標を確認
