1. モデル評価の基本概念

学習時間の目安: 30分
このセクションの目標: なぜモデルを評価するのか、評価の落とし穴（過学習・過少学習）を理解し、データ分割の正しい方法を習得する。

1.1 なぜモデル評価が重要か

機械学習モデルを構築した後、そのモデルが未知のデータに対してどれほどうまく機能するかを知ることが最重要課題です。学習データに対して完璧に動くモデルが、現実のデータには全く使えない、というケースが頻繁に起こります。

具体例: 過去の天気データ10,000件で学習した「明日の天気予測モデル」が、訓練データでは99%の精度を示した。しかし実際の予測では60%しか当たらなかった。これは過学習（Overfitting）の典型例。

1.3 データ分割の基本

モデルを公平に評価するためには、学習に使っていないデータで評価する必要があります。

全データセット（例: 10,000件）
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  訓練データ（Train）  │  検証データ（Val）  │  テスト │
│       70%             │       15%           │  15%    │
│    7,000件            │     1,500件         │ 1,500件 │
│                                                         │
│  モデルの学習に使う   │ ハイパーパラメータ  │  最終   │
│                       │   チューニングに使う │  評価   │
└─────────────────────────────────────────────────────────┘

注意: テストデータは最後の1回だけ使う！何度も使うと「情報漏洩」が起きる。
重要: テストデータはモデル開発中は絶対に見てはいけません。最終的な性能報告のためだけに1回使います。検証データを繰り返し使うと、間接的に検証データに過学習してしまいます。

1.4 バイアス・バリアンスのトレードオフ

バイアスとバリアンスの定義

  • バイアス（Bias）: モデルの予測が真の値から系統的にずれる量。高バイアス = 単純すぎるモデル
  • バリアンス（Variance）: 異なる訓練データでモデルがどれほど変動するか。高バリアンス = 複雑すぎるモデル
  • 総誤差 = バイアス² + バリアンス + ノイズ
総誤差 = Bias² + Variance + Irreducible Noise
目標: Bias と Variance のバランスを取り、総誤差を最小化する

2. 分類モデルのメトリクス

学習時間の目安: 45分
このセクションの目標: 混同行列から始まり、精度・適合率・再現率・F1スコア・AUC-ROCまで、分類評価の全メトリクスを数式と具体例で理解する。

2.1 混同行列（Confusion Matrix）

分類モデルの評価の基礎は混同行列です。予測結果と実際のクラスの組み合わせを表にまとめたもので、ここから全ての分類メトリクスが導出されます。

例: がん検診モデル（陽性=がんあり、陰性=がんなし）
混同行列 予測
陽性（Positive） 陰性（Negative）
実際 陽性（Positive） TP = 90
真陽性（正しく陽性）		 FN = 10
偽陰性（見逃し）
陰性（Negative） FP = 20
偽陽性（誤警報）		 TN = 880
真陰性（正しく陰性）

合計: 1,000件 TP=90, FN=10, FP=20, TN=880

4つの基本値の意味

  • TP（True Positive / 真陽性）: がんを正しくがんと判定 → 90件
  • TN（True Negative / 真陰性）: 健康を正しく健康と判定 → 880件
  • FP（False Positive / 偽陽性）: 健康をがんと誤判定（Type I Error）→ 20件
  • FN（False Negative / 偽陰性）: がんを健康と誤判定（Type II Error）→ 10件

2.2 主要メトリクスの計算

Accuracy
（精度）

(TP+TN)/(全体)
例: (90+880)/1000 = 97%

Precision
（適合率）

TP/(TP+FP)
例: 90/(90+20) = 81.8%

Recall
（再現率）

TP/(TP+FN)
例: 90/(90+10) = 90%

F1 Score

2×P×R/(P+R)
例: 2×0.818×0.9/1.718 ≈ 85.7%

2.3 各メトリクスの使い分け

Accuracyの罠（不均衡データ問題）: がん検診で1,000件中10件しかがん患者がいない場合、「全員健康」と予測するだけで Accuracy = 99%。しかしこのモデルは全くの無意味。不均衡データでは Accuracy だけで判断してはいけない。

2.4 Precision-Recall のトレードオフ

閾値を下げる
(より積極的に陽性判定)

Recall 上昇
Precision 低下
⇄
閾値を上げる
(より慎重に陽性判定)

Precision 上昇
Recall 低下
ポイント: モデルの出力は確率（例: 0.7）であり、「陽性か陰性か」の判定は閾値（通常0.5）で決まります。閾値を変えると Precision と Recall が反対方向に動きます。ビジネス要件に応じて最適な閾値を選択してください。

2.5 ROC曲線とAUC

  TPR（再現率）
  1.0 |    .-------------- 完璧なモデル (AUC=1.0)
      |  .-'
      | /  <---- 良いモデル (AUC=0.85前後)
  0.5 |/
      |  <----- ランダム予測 (AUC=0.5, 対角線)
      |/
  0.0 +---------------------
      0.0        0.5        1.0
                FPR（偽陽性率 = FP/(FP+TN)）

  AUC（Area Under the Curve）= ROC曲線の下の面積
  AUC = 1.0: 完璧  |  AUC = 0.5: ランダムと同じ  |  AUC < 0.5: 悪い

AUCの解釈

AUC = 0.85 は「ランダムに選んだ陽性サンプルが、ランダムに選んだ陰性サンプルよりも高いスコアを得る確率が85%」という意味です。

  • AUC 0.9以上: 非常に良い
  • AUC 0.8〜0.9: 良い
  • AUC 0.7〜0.8: 普通
  • AUC 0.6〜0.7: 改善が必要
  • AUC 0.6未満: 問題あり
AWS試験ポイント: Amazon SageMaker の Binary Classification では AUC が標準的な評価メトリクスとして提供されます。不均衡なデータセットでは Accuracy より AUC や F1 Score を重視すること。

3. 回帰モデルのメトリクス

学習時間の目安: 30分
このセクションの目標: 回帰タスク（連続値予測）で使うMAE・MSE・RMSE・R²を理解し、それぞれの特性と使い分けを習得する。

3.1 主要な回帰メトリクス

例: 住宅価格予測（単位: 万円）
実際の価格: [3000, 2500, 4000, 1800, 3500]
予測した価格: [2900, 2700, 3800, 2100, 3400]
誤差: [-100, +200, -200, +300, -100]
メトリクス 計算式 上記例での値 特徴
MAE
平均絶対誤差		 Σ|y - ŷ| / n (100+200+200+300+100)/5
= 180万円		 解釈しやすい
外れ値に頑健
MSE
平均二乗誤差		 Σ(y - ŷ)² / n (10000+40000+40000+90000+10000)/5
= 38000万円²		 外れ値を強く罰する
単位が二乗になり解釈困難
RMSE
二乗平均平方根誤差		 √MSE √38000 ≈ 195万円 元の単位で解釈可能
外れ値に敏感
R²
決定係数		 1 - SS_res/SS_tot 0〜1の値
1に近いほど良い		 モデルがデータの変動を

3.2 MAEとRMSEの選択基準

MAEを選ぶとき

  • 外れ値が多いデータ
  • 外れ値を特別扱いしたくない
  • ビジネス担当者への説明が必要
  • 例: 需要予測、在庫管理

RMSEを選ぶとき

  • 大きな誤差を特に避けたい
  • 外れ値を強く罰したい
  • 数学的な最適化が必要
  • 例: 気温予測、株価予測

3.3 決定係数（R²）の理解

R² = 1 - (残差平方和 / 全変動)
= 1 - SS_residuals / SS_total

R² = 0.85 → モデルは目的変数の変動の85%を説明できる
R²の値 解釈 例
R² = 1.0 完璧な予測（過学習の可能性） —
R² = 0.9以上 非常に良い 精密な物理実験
R² = 0.7〜0.9 良い 住宅価格予測
R² = 0.5〜0.7 普通 株価予測
R² 0.5未満 改善が必要 —
R² 負（マイナス） 平均値より悪い予測 完全に失敗したモデル
R²の注意点: R²は特徴量を追加するだけで必ず上がります（または維持）。そのため、特徴量数を考慮した「調整済みR²（Adjusted R²）」を使うことが推奨されます。特徴量が多いモデル同士を比較する際には調整済みR²が適切です。

4. 生成AIモデルの評価

学習時間の目安: 30分
このセクションの目標: LLMや生成系AIの評価は従来のMLと異なる。BLEU、ROUGE、人間評価、LLMによる評価など、生成AIに特化した評価手法を理解する。

4.1 生成AIの評価が難しい理由

従来MLとの違い: 画像分類は「猫か犬か」と明確に正解が一つ。しかし文章生成では「良い返答」は無数に存在します。「今日の天気は？」への回答「晴れです」と「本日は晴天です」は、どちらも正解ですが異なる文字列です。この問題に対応するため、生成AI専用のメトリクスが開発されました。

4.3 LLM-as-a-Judge（LLMによる評価）

近年、別のLLM（GPT-4やClaudeなど）を使って生成AIの出力を評価する「LLM-as-a-Judge」手法が注目されています。

LLM評価の利点:
  • 自動化が可能（人間評価より安価）
  • N-gramマッチより意味を理解
  • 多面的な評価基準を設定可能
  • スケールしやすい
LLM評価の注意点:
  • 評価LLM自体のバイアス
  • 「長い回答を好む」傾向
  • 自分と似たスタイルを好む
  • コストが高い

5. 交差検証とモデル選択

学習時間の目安: 30分
このセクションの目標: 少ないデータでも信頼性の高い評価ができるK分割交差検証を理解し、正しいモデル選択の方法を習得する。

5.1 なぜ交差検証が必要か

データを単純に「訓練80%、テスト20%」に分けると、たまたまテストデータが偏っていた場合に評価が不安定になります。交差検証はこの問題を解決します。

問題の例: 同じモデルを3回評価したところ、Accuracy が 75%, 88%, 82% とバラバラ。これはテストデータの分割方法に依存しており、モデルの真の性能が分からない。

5.2 K分割交差検証（K-Fold Cross Validation）

データ全体を K個に分割（例: K=5）

フォールド1: [テスト] [訓練][訓練][訓練][訓練]  → Score1 = 82%
フォールド2: [訓練][テスト] [訓練][訓練][訓練]  → Score2 = 85%
フォールド3: [訓練][訓練][テスト] [訓練][訓練]  → Score3 = 79%
フォールド4: [訓練][訓練][訓練][テスト] [訓練]  → Score4 = 88%
フォールド5: [訓練][訓練][訓練][訓練][テスト]   → Score5 = 84%

最終スコア: (82+85+79+88+84)/5 = 83.6% ± 3.2%
交差検証の利点: 全データをテストに使えるため、より信頼性の高い評価が得られます。また、スコアの標準偏差（±3.2%）でモデルの安定性も確認できます。

5.3 交差検証の種類

手法 特徴 適用場面 計算コスト
K-Fold データをK分割して評価 一般的な用途（K=5〜10） 中
Stratified K-Fold 各フォールドでクラス比率を
維持する		 不均衡データの分類 中

(K=データ数)		 データが非常に少ない場合 非常に高

時系列データの注意: 株価・気温などの時系列データにランダムK-Foldを適用してはいけません。将来のデータで訓練し、過去のデータでテストするという「未来漏洩」が起きます。必ず時系列分割を使用してください。

5.4 モデル選択の基準

考慮要素 詳細
汎化性能 交差検証スコアが高く、かつ安定している（分散が小さい）
ビジネス要件 精度 vs 再現率のトレードオフ、推論速度、解釈可能性
計算コスト 訓練時間、推論時間、メモリ使用量
統計的有意性 2つのモデルの差が統計的に有意かを t検定で確認
Occam's Razor 同等の性能なら、シンプルなモデルを選ぶ

モデルの性能だけでなく、公平性（Fairness）の評価もAWS試験の重要トピックです。

バイアスメトリクス 説明 例
Class Imbalance (CI) 訓練データ内のクラス不均衡の度合い 男性データ70%、女性30% → CI = 0.4
DPL (Difference in
Positive Proportions)		 グループ間での陽性予測率の差 男性の採用率80%、女性60%
Disparate Impact (DI) 不利グループの陽性率 / 有利グループの陽性率 DI が0.8未満で差別的とみなす（米国法）
AWS試験ポイント:
  • SageMaker Clarify = バイアス検出 + モデル説明可能性（SHAP）
  • SageMaker Model Monitor = 本番モデルの継続的な品質監視
  • Bedrock = 生成AIモデルの評価（ROUGE, BERTScore）
  • データドリフトとコンセプトドリフトの違いを理解する

6.4 データドリフトとコンセプトドリフト

データドリフト（Data Drift）

入力データの統計的分布が変化すること。

例: ECサイトのユーザー年齢層が変化し、「平均年齢35歳」だったものが「平均年齢25歳」になった。

対策: 入力データの統計を監視し、閾値を超えたらアラート。

コンセプトドリフト（Concept Drift）

入力と出力の関係性が変化すること。

例: コロナ前に「旅行」という単語 → 高収益だったが、コロナ後は同じ単語でも低収益に。

対策: 定期的なモデルの再訓練と評価。

7. 試験対策まとめ・演習問題

学習時間の目安: 10分

7.1 重要メトリクス早見表

タスク種別 主要メトリクス 使用場面
二値分類（均衡） Accuracy, AUC スパム分類、健全なデータ
二値分類（不均衡） F1 Score, AUC, Precision, Recall 詐欺検知、医療診断
多クラス分類 Weighted F1, Macro F1 感情分析、文書分類
回帰（外れ値あり） MAE, Huber Loss 需要予測
回帰（外れ値なし） RMSE, R² 気温予測、価格予測
機械翻訳 BLEU, METEOR 翻訳品質評価
テキスト要約 ROUGE-1, ROUGE-2, ROUGE-L 要約品質評価
LLM全般 Perplexity, BERTScore, Human Eval 基盤モデル評価

7.2 重要公式まとめ

Accuracy = (TP + TN) / (TP + TN + FP + FN) Precision = TP / (TP + FP) ← 「陽性と言ったものが正しい割合」 Recall = TP / (TP + FN) ← 「実際の陽性を見つけた割合」 F1 Score = 2 × (Precision × Recall) / (Precision + Recall) MAE = Σ|y - ŷ| / n RMSE = √(Σ(y - ŷ)² / n) R² = 1 - SS_residuals / SS_total

7.3 よく出る試験パターン

1

不均衡データ問題

「データの99%が陰性」「詐欺検知」「がん検診」→ Accuracy は使えない。F1、AUC、Precision/Recall を使う。

2

見逃しが致命的な場面

「がん見逃し」「テロリスト見逃し」→ Recall（再現率）を最大化。FNを最小化。

3

誤検知が致命的な場面

「重要メールをスパム判定」「正常を異常と判断」→ Precision（適合率）を最大化。FPを最小化。

4

生成AIの評価

翻訳 → BLEU、要約 → ROUGE、LLM全般 → Perplexity + 人間評価

5

AWSサービスとメトリクス

Clarify → バイアス検出、SHAP。Model Monitor → ドリフト検知。Bedrock → ROUGE, BERTScore評価

演習問題（AIF-C01 スタイル）

問1. ある医療診断AIモデルで、がんの見逃しを最小化することが最優先事項です。最も重視すべきメトリクスはどれですか？

A) Accuracy
B) Precision（適合率）
C) Recall（再現率）★正解★
D) BLEU Score

解説: がんの見逃し = FNを最小化 = Recall（TP/(TP+FN)）を最大化。Recall が正解。BLEU は自然言語評価に使われる。

問2. 機械翻訳システムの品質を自動評価するために最も適したメトリクスはどれですか？

A) ROUGE-L
B) BLEU ★正解★
C) RMSE
D) AUC-ROC

解説: BLEUは機械翻訳評価の標準メトリクス。ROUGEはテキスト要約評価に使用。RMSEは回帰評価。AUCは分類評価。

問3. 本番環境で稼働中のモデルが突然性能劣化し始めました。「入力データの分布は変わっていないが、入力と正解ラベルの関係が変化した」という状況を何と呼びますか？

A) データドリフト
B) コンセプトドリフト ★正解★
C) 過学習
D) クラス不均衡

解説: コンセプトドリフトは入力→出力の関係性が変化すること。データドリフトは入力データの分布変化。SageMaker Model Monitorで検知可能。

問4. 1,000件中10件しか不正取引がないデータで訓練した詐欺検知モデルのAccuracyが99%でした。このモデルについて正しい説明はどれですか？

A) このモデルは非常に優秀で本番環境に即デプロイできる
B) 全件を正常と判断するだけで99%になり、詐欺を検知できていない可能性がある ★正解★
C) Accuracyが高いため他のメトリクスを確認する必要はない
D) モデルが過少学習している証拠である

解説: 不均衡データでAccuracyは無意味になりうる。全件「正常」と予測するだけでAccuracy 99%。F1 ScoreやRecallを確認すること。
